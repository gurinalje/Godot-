# RPG Growth System

> **Status**: Designed
> **Author**: User + Agents
> **Last Updated**: 2026-06-03
> **Implements Pillar**: Pillar 4 (Persistent Growth)

## Overview

RPG成长系统是《命运卡牌局》的角色成长层，负责管理玩家角色的属性成长和技能解锁。系统连接角色属性系统和技能树系统，为玩家提供完整的RPG成长体验。

## Player Fantasy

**"我的角色在不断变强，解锁新的能力"**

玩家应该感受到：
- **成长感**：角色在不断升级变强
- **成就感**：解锁新技能的满足感
- **策略性**：选择哪些技能优先解锁
- **投入感**：为成长投入资源的满足感
- **期待感**：期待下一次升级的能力

**参考体验**：暗黑破坏神的角色成长 + 最终幻想的技能树 + 杀戮尖塔的遗物系统

## Detailed Design

### Core Rules

**角色成长**：
- 角色通过战斗和任务获得经验值
- 经验值达到阈值后升级
- 每次升级获得属性点和技能点
- 属性点可分配到6个基础属性
- 技能点可解锁技能树中的技能

**属性成长**：
- **力量（STR）**：影响物理伤害
- **敏捷（DEX）**：影响暴击率和闪避
- **智力（INT）**：影响魔法伤害和能量上限
- **体质（CON）**：影响生命值和防御
- **感知（PER）**：影响发现和命中
- **幸运（LCK）**：影响掉落和暴击伤害

**技能树**：
- 每个元素有独立的技能树
- 技能树有前置技能要求
- 技能有主动和被动之分
- 技能有等级限制

**成长限制**：
- 最高等级：100级
- 属性上限：149（基础99+装备50）
- 技能点数量：有限（每级1点）

### States and Transitions

| 当前状态 | 触发条件 | 目标状态 | 说明 |
|----------|----------|----------|------|
| 1级 | 获得经验值 | 升级中 | 开始升级 |
| 升级中 | 经验值达到阈值 | 已升级 | 升级成功 |
| 已升级 | 分配属性点 | 属性分配中 | 分配属性点 |
| 属性分配中 | 分配完成 | 已升级 | 完成分配 |
| 已升级 | 解锁技能 | 技能解锁中 | 解锁技能 |
| 技能解锁中 | 解锁完成 | 已升级 | 完成解锁 |
| 已满级 | 无法升级 | 已满级 | 达到最高等级 |

### Interactions with Other Systems

| 交互系统 | 方向 | 接口 | 说明 |
|----------|------|------|------|
| 角色属性系统 | 输出 | get_player_stats() | 获取玩家属性 |
| 技能树系统 | 输出 | get_skill_tree() | 获取技能树 |
| 卡牌战斗系统 | 输入 | get_battle_rewards() | 获取战斗奖励 |
| 存档系统 | 输出 | save_growth_state() | 保存成长状态 |
| UI系统 | 输出 | update_growth_ui() | 更新成长界面 |

## Formulas

**经验需求公式**：

`xp_required = 100 × (level ^ 1.5)`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| level | - | int | 1-100 | 当前等级 |

**输出范围**：100到100000
**示例**：level=10 → xp_required=100×31.62=3162

**属性点分配公式**：

`attribute_points = 3 × (level - 1)`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| level | - | int | 1-100 | 当前等级 |

**输出范围**：0到297
**示例**：level=20 → attribute_points=3×19=57

**技能点分配公式**：

`skill_points = level - 1`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| level | - | int | 1-100 | 当前等级 |

**输出范围**：0到99
**示例**：level=30 → skill_points=29

## Edge Cases

| # | 边界情况 | 处理方式 |
|---|----------|----------|
| 1 | **经验值溢出** | 验证经验值合法性 |
| 2 | **属性点不足** | 提示"属性点不足"，阻止分配 |
| 3 | **技能点不足** | 提示"技能点不足"，阻止解锁 |
| 4 | **属性已达上限** | 提示"属性已达上限"，阻止分配 |
| 5 | **技能前置未满足** | 提示"需要前置技能"，阻止解锁 |
| 6 | **等级已达上限** | 提示"已达最高等级"，阻止升级 |
| 7 | **成长状态异常** | 验证状态合法性，修复异常 |
| 8 | **属性分配错误** | 提供重置机制（消耗资源） |

## Tuning Knobs

| 调优参数 | 默认值 | 安全范围 | 极端行为 |
|----------|--------|----------|----------|
| 最高等级 | 100 | 50-150 | 太低=成长受限，太高=过于复杂 |
| 经验需求指数 | 1.5 | 1.2-2.0 | 太慢=无挑战，太快=过于困难 |
| 每级属性点 | 3 | 2-5 | 太少=成长受限，太多=过于强大 |
| 每级技能点 | 1 | 1-2 | 太少=技能受限，太多=过于强大 |
| 属性上限 | 149 | 99-199 | 太低=成长受限，太高=过于强大 |
| 技能树深度 | 5 | 3-7 | 太浅=策略不足，太深=过于复杂 |
| 属性重置消耗 | 1000金币 | 500-2000 | 太低=无代价，太高=过于严格 |
| 技能重置消耗 | 500金币 | 200-1000 | 太低=无代价，太高=过于严格 |

## UI Requirements

**RPG成长界面**：
```
┌─────────────────────────────────────────┐
│  角色信息（名称/等级/经验值）           │
├─────────────────────────────────────────┤
│  属性面板（6个基础属性 + 分配按钮）     │
│  STR: 50 [+]  DEX: 30 [+]  INT: 40 [+] │
│  CON: 35 [+]  PER: 25 [+]  LCK: 20 [+] │
├─────────────────────────────────────────┤
│  技能树（技能图标 + 连接线）            │
│  ┌─────┐    ┌─────┐    ┌─────┐         │
│  │ 技能1│────│ 技能2│────│ 技能3│         │
│  └─────┘    └─────┘    └─────┘         │
├─────────────────────────────────────────┤
│  操作按钮（分配/解锁/重置/返回）        │
└─────────────────────────────────────────┘
```

**交互规则**：
- 点击属性+按钮 → 分配属性点
- 点击技能图标 → 解锁技能
- 点击重置按钮 → 重置属性/技能
- 拖拽技能树 → 移动视图
- 缩放 → 放大/缩小技能树
