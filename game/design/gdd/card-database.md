# 卡牌数据库 (Card Database)

> **Status**: In Design  
> **Author**: [user + agents]  
> **Last Updated**: 2026-06-03  
> **Implements Pillar**: 选择即规则、探索奖励好奇心、持久的成长感

---

## Overview

**卡牌数据库**是游戏的卡牌数据定义层，负责存储和管理所有卡牌的属性、效果、美术资源引用等核心数据。它是所有卡牌相关系统（伤害计算、状态效果、Combo连锁、元素系统、卡组构筑、卡牌升级）的基础数据源。

系统使用 Godot 的 Resource 系统（.tres 文件）作为数据存储格式，每张卡牌定义为一个独立的 CardData 资源。卡牌数据库提供查询接口，支持按类型、元素、稀有度、费用等条件检索卡牌。

作为基础设施层，卡牌数据库不直接面向玩家，而是通过卡牌战斗、卡组管理等上层系统间接影响玩家体验。

## Player Fantasy

> 玩家通过卡牌数据库感受到"我的收藏在不断壮大"——每获得一张新卡牌，都像是打开命运之书的新一页。卡牌数据库是这种收藏感和成长感的底层支撑。

**幻想锚点：**
- **情感目标**：期待、发现、满足
- **玩家时刻**：获得新卡牌时的"叮！"瞬间
- **参考感受**：炉石传说开卡包的兴奋感，杀戮尖塔获得稀有卡的惊喜感

## Detailed Design

### Core Rules

**卡牌数据结构（CardData Resource）：**

| 字段 | 类型 | 必填 | 描述 |
|------|------|------|------|
| `id` | String | ✅ | 唯一标识符，格式：`card_[类型]_[序号]` |
| `name` | String | ✅ | 卡牌显示名称 |
| `description` | String | ✅ | 卡牌效果描述文本 |
| `card_type` | CardType枚举 | ✅ | SUMMON / DIRECT_DAMAGE / ENVIRONMENT / BUFF_DEBUFF |
| `cost` | int | ✅ | 能量消耗，范围：0-10 |
| `effects` | Array[CardEffect] | ✅ | 效果列表，至少1个 |
| `rarity` | Rarity枚举 | ✅ | COMMON / UNCOMMON / RARE / LEGENDARY |
| `element` | Element枚举 | 可选 | FIRE / WATER / EARTH / WIND / LIGHTNING / NONE |
| `artwork` | Texture2D | 可选 | 卡牌插画 |
| `keywords` | Array[String] | 可选 | 关键词标签，用于搜索和过滤 |

**卡牌效果结构（CardEffect Resource）：**

| 字段 | 类型 | 描述 |
|------|------|------|
| `effect_type` | EffectType枚举 | DAMAGE / HEAL / SUMMON / BUFF / DEBUFF / ENVIRONMENT_CHANGE |
| `value` | int | 效果数值 |
| `target` | TargetType枚举 | SELF / ENEMY / ALL_ENEMIES / ALL_ALLIES / RANDOM |
| `duration` | int | 持续回合数，0表示即时效果 |
| `condition` | String | 触发条件表达式（可选） |

**四大卡牌类型规则：**

| 类型 | 核心规则 | 设计意图 |
|------|----------|----------|
| **召唤类** | 召唤生物到战场，有生命值和攻击力 | 创造场面优势 |
| **直接伤害类** | 对目标造成即时伤害 | 快速清除或斩杀 |
| **环境阵地类** | 改变战场环境，影响所有单位 | 策略深度，控制战场 |
| **增益减益类** | 给予Buff/Debuff，可叠加 | 组合Combo的关键 |

### States and Transitions

**卡牌生命周期状态机：**

| 状态 | 描述 | 可转换到 |
|------|------|----------|
| **未解锁** | 玩家尚未获得此卡牌 | 已解锁 |
| **已解锁** | 已获得但未装备 | 已装备 |
| **已装备** | 在卡组中，可使用 | 已使用、已移除 |
| **已使用** | 本回合已打出 | 已装备（下回合） |
| **已移除** | 从卡组移除 | 已解锁 |

### Interactions with Other Systems

| 系统 | 数据流向 | 接口描述 |
|------|----------|----------|
| **伤害计算系统** | 输出 | 提供卡牌伤害值、元素类型 |
| **状态效果系统** | 输出 | 提供Buff/Debuff效果数据 |
| **Combo连锁系统** | 输出 | 提供卡牌ID和效果，用于连锁判定 |
| **元素系统** | 输出 | 提供卡牌元素属性 |
| **卡组构筑系统** | 双向 | 查询可用卡牌、更新卡组配置 |
| **卡牌升级系统** | 双向 | 查询卡牌基础属性、更新升级后属性 |
| **存档系统** | 输出 | 提供卡牌收藏数据用于保存 |
| **UI系统** | 输出 | 提供卡牌显示信息 |

## Formulas

### 卡牌能量消耗与效果价值公式

**效果价值公式：**

`effect_value = base_value × rarity_multiplier × element_modifier`

**变量：**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_value | V_b | int | 1-20 | 基础效果数值 |
| rarity_multiplier | M_r | float | 1.0-2.0 | 稀有度倍率 |
| element_modifier | M_e | float | 0.8-1.2 | 元素加成/削弱 |

**稀有度倍率表：**
| 稀有度 | 倍率 |
|--------|------|
| COMMON | 1.0 |
| UNCOMMON | 1.3 |
| RARE | 1.6 |
| LEGENDARY | 2.0 |

**输出范围：** 0.8 到 40.0（正常游戏范围）
**示例：** 基础伤害5 × RARE(1.6) × FIRE(1.1) = 8.8 → 取整为9

### 能量消耗平衡公式

**能量消耗公式：**

`energy_cost = ceil(effect_value / cost_efficiency)`

**变量：**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| effect_value | V_e | float | 0.8-40.0 | 效果价值（来自上一公式） |
| cost_efficiency | C_e | float | 2.0-5.0 | 每点能量的效果值 |

**输出范围：** 0 到 10（能量消耗上限）
**示例：** 效果价值8.8 ÷ 效率3.0 = 2.93 → 向上取整为3

## Edge Cases

- **如果卡牌ID重复**：系统启动时输出错误日志，使用第一个加载的卡牌，忽略重复项。
- **如果卡牌效果列表为空**：该卡牌标记为无效，不出现在玩家可获得列表中。
- **如果能量消耗超过10**：强制设为10，并输出警告日志。
- **如果效果数值为0或负数**：该效果不执行，但卡牌仍然可以使用（可能有其他效果）。
- **如果卡牌数据文件损坏**：使用备份数据或默认卡牌，提示玩家数据异常。
- **如果卡牌插画缺失**：显示默认占位图，不影响游戏功能。
- **如果卡牌被引用但已从数据库移除**：存档加载时跳过该卡牌，保留其他数据。

## Dependencies

### 硬依赖（无）
卡牌数据库是基础层系统，无硬依赖。

### 软依赖（无）
无软依赖。

### 被依赖系统
| 系统 | 依赖类型 | 描述 |
|------|----------|------|
| 伤害计算系统 | 硬依赖 | 需要卡牌伤害值和元素类型 |
| 状态效果系统 | 硬依赖 | 需要Buff/Debuff效果数据 |
| Combo连锁系统 | 硬依赖 | 需要卡牌ID和效果用于连锁判定 |
| 元素系统 | 硬依赖 | 需要卡牌元素属性 |
| 卡组构筑系统 | 硬依赖 | 需要查询可用卡牌 |
| 卡牌升级系统 | 硬依赖 | 需要查询卡牌基础属性 |

## Tuning Knobs

| 调优参数 | 默认值 | 范围 | 描述 | 极端值影响 |
|----------|--------|------|------|------------|
| `max_energy_cost` | 10 | 5-20 | 卡牌最大能量消耗 | 过高：破坏平衡；过低：限制设计空间 |
| `rarity_multiplier_common` | 1.0 | 0.8-1.2 | 普通卡倍率 | 影响所有普通卡的价值 |
| `rarity_multiplier_uncommon` | 1.3 | 1.0-1.5 | 稀有卡倍率 | 影响稀有卡的价值 |
| `rarity_multiplier_rare` | 1.6 | 1.2-2.0 | 史诗卡倍率 | 影响史诗卡的价值 |
| `rarity_multiplier_legendary` | 2.0 | 1.5-3.0 | 传说卡倍率 | 影响传说卡的价值 |
| `cost_efficiency_base` | 3.0 | 2.0-5.0 | 基础能量效率 | 过高：卡牌太便宜；过低：卡牌太贵 |
| `element_modifier_range` | 0.2 | 0.1-0.3 | 元素加成范围 | 影响元素克制的强度 |
| `max_cards_per_deck` | 40 | 20-50 | 卡组最大卡牌数 | 过高：策略性降低；过低：构筑受限 |

## Visual/Audio Requirements

**视觉需求：**
- 卡牌界面需要显示卡牌插画、名称、类型、能量消耗、效果描述
- 不同稀有度使用不同边框样式（普通=灰色，稀有=蓝色，史诗=紫色，传说=金色）
- 不同元素使用不同颜色标识（火=红，水=蓝，土=黄，风=绿，雷=紫）

**音频需求：**
- 获得新卡牌时播放"叮！"音效
- 卡牌翻转时播放翻页声
- 稀有卡获得时播放特殊音效

## UI Requirements

**卡牌浏览界面：**
- 显示所有已解锁卡牌的列表
- 支持按类型、元素、稀有度筛选
- 支持按名称、费用、稀有度排序
- 点击卡牌显示详细信息

**卡牌详情界面：**
- 显示卡牌插画、名称、类型、能量消耗
- 显示效果描述文本
- 显示稀有度和元素标识
- 显示卡牌状态（已解锁/已装备/已使用）

## Acceptance Criteria

| 编号 | 验收标准 | 测试方法 |
|------|----------|----------|
| AC-1 | 卡牌数据库包含所有四种类型的卡牌 | 查询数据库，验证每种类型至少有1张卡牌 |
| AC-2 | 卡牌数据结构完整，所有必填字段都有值 | 遍历所有卡牌，检查必填字段不为空 |
| AC-3 | 卡牌效果可以被伤害计算系统正确读取 | 调用接口，验证返回值与预期一致 |
| AC-4 | 卡牌效果可以被状态效果系统正确读取 | 调用接口，验证返回值与预期一致 |
| AC-5 | 卡牌ID全局唯一，无重复 | 遍历所有卡牌，检查ID唯一性 |
| AC-6 | 卡牌能量消耗在0-10范围内 | 遍历所有卡牌，检查cost字段范围 |
| AC-7 | 卡牌效果列表至少包含1个效果 | 遍历所有卡牌，检查effects数组长度 |
| AC-8 | 卡牌稀有度和元素使用正确的枚举值 | 遍历所有卡牌，检查枚举值有效性 |
| AC-9 | 卡牌数据可以被存档系统正确保存和加载 | 保存后加载，验证数据一致性 |
| AC-10 | 卡牌浏览界面可以正确显示所有卡牌 | 打开界面，验证卡牌显示正确 |
