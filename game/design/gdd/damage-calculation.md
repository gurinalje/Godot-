# 伤害计算系统

> **Status**: In Design
> **Author**: [user + agents]
> **Last Updated**: 2026-06-03
> **Implements Pillar**: 支柱2 — 爽快的策略深度

## Overview

伤害计算系统是游戏战斗的核心数学引擎，负责将卡牌效果转化为实际伤害数值。它读取卡牌数据库的基础伤害值和元素类型，结合角色属性系统的力量/智力等属性，经过公式计算输出最终伤害。玩家通过这个系统感受到"我的卡牌选择和属性分配产生了有意义的结果"——一张精心选择的卡牌配合正确的属性加成，可以造成毁灭性的伤害。

## Player Fantasy

"我的选择产生了有意义的伤害" — 玩家应该感受到：
- **掌控感**：我选择的卡牌和属性分配直接影响伤害输出
- **爽快感**：看到大数字跳出时的满足感
- **策略深度**：理解元素克制和属性加成后，能打出更高伤害

**情感锚点**：像炉石传说中一张法术打出20点伤害的爽快感——"我算对了，这一击致命！"

## Detailed Design

### Core Rules

1. **基础伤害计算**：
   - 读取卡牌的基础伤害值（CardEffect.value）
   - 读取卡牌的元素类型（CardData.element）

2. **属性加成计算**：
   - 物理伤害：基于力量（STR）
   - 法术伤害：基于智力（INT）
   - 命中率：基于感知（PER）
   - 暴击率：基于敏捷（DEX）
   - 暴击伤害：基于幸运（LCK）

3. **元素克制系统**：
   - 火 > 风 > 土 > 水 > 火（循环克制）
   - 雷独立，对所有元素有加成
   - 克制倍率：1.2x（攻击方优势）
   - 被克制倍率：0.8x（攻击方劣势）

4. **暴击系统**：
   - 基础暴击率：5%
   - 暴击率加成：DEX * 0.5%
   - 最大暴击率：50%
   - 基础暴击伤害：150%
   - 暴击伤害加成：LCK * 1%
   - 最大暴击伤害：300%

5. **防御计算**：
   - 物理防御：基于体质（CON）
   - 法术防御：基于智力（INT）
   - 减伤公式：damage_reduction = defense / (defense + 100)

6. **最终伤害公式**：
   ```
   final_damage = (base_damage + attribute_bonus) × element_modifier × crit_multiplier × (1 - damage_reduction)
   ```

### States and Transitions

| 状态 | 描述 | 触发条件 |
|------|------|----------|
| **待计算** | 伤害计算开始 | 卡牌打出 |
| **计算中** | 执行伤害公式 | 读取属性和卡牌数据 |
| **暴击判定** | 检查是否暴击 | 随机数 < 暴击率 |
| **元素判定** | 检查元素克制 | 比较攻击和防御元素 |
| **输出** | 输出最终伤害 | 计算完成 |

**状态转换规则**：
- 待计算 → 计算中：卡牌打出
- 计算中 → 暴击判定：读取DEX和LCK
- 暴击判定 → 元素判定：暴击结果确定
- 元素判定 → 输出：元素倍率确定
- 输出 → 待计算：伤害应用完成

### Interactions with Other Systems

| 系统 | 数据流向 | 接口描述 |
|------|----------|----------|
| **卡牌数据库** | 输入 | 读取卡牌基础伤害值和元素类型 |
| **角色属性系统** | 输入 | 读取STR/DEX/INT/CON/PER/LCK属性值 |
| **召唤物系统** | 输出 | 提供召唤物伤害计算结果 |
| **卡牌战斗系统** | 输出 | 提供卡牌伤害计算结果 |
| **状态效果系统** | 输入 | 读取Buff/Debuff对伤害的加成/削弱 |
| **元素系统** | 输入 | 读取元素克制关系 |

## Formulas

### 基础伤害公式

`base_damage = card_effect_value + (attribute_bonus × attribute_scaling)`

**变量：**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| card_effect_value | V_c | int | 1-20 | 卡牌基础伤害值（来自CardEffect.value） |
| attribute_bonus | A_b | int | 0-149 | 角色属性值（STR或INT） |
| attribute_scaling | S_a | float | 0.1-0.5 | 属性加成系数 |

**输出范围：** 1 到 94（正常游戏范围）
**示例：** 卡牌伤害5 + (力量50 × 0.3) = 5 + 15 = 20

### 元素克制公式

`element_modifier = base_modifier × (1 + element_advantage)`

**变量：**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_modifier | M_b | float | 1.0 | 基础倍率 |
| element_advantage | E_a | float | -0.2 to +0.2 | 元素优势/劣势 |

**元素克制表：**
| 攻击元素 | 防御元素 | 优势值 |
|----------|----------|--------|
| 火 | 风 | +0.2 |
| 风 | 土 | +0.2 |
| 土 | 水 | +0.2 |
| 水 | 火 | +0.2 |
| 雷 | 所有 | +0.1 |
| 被克制 | — | -0.2 |

**输出范围：** 0.8 到 1.2（正常游戏范围）
**示例：** 火攻击风防御 = 1.0 × (1 + 0.2) = 1.2

### 暴击率公式

`crit_rate = base_crit + (DEX × dex_crit_scaling)`

**变量：**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_crit | C_b | float | 0.05 | 基础暴击率（5%） |
| DEX | DEX | int | 0-149 | 敏捷属性值 |
| dex_crit_scaling | S_d | float | 0.005 | 敏捷对暴击率的加成系数 |

**输出范围：** 0.05 到 0.50（5%到50%）
**示例：** 基础5% + (敏捷60 × 0.5%) = 5% + 30% = 35%

### 暴击伤害公式

`crit_damage = base_crit_damage + (LCK × lck_crit_scaling)`

**变量：**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_crit_damage | CD_b | float | 1.5 | 基础暴击伤害（150%） |
| LCK | LCK | int | 0-149 | 幸运属性值 |
| lck_crit_scaling | S_l | float | 0.01 | 幸运对暴击伤害的加成系数 |

**输出范围：** 1.5 到 3.0（150%到300%）
**示例：** 基础150% + (幸运40 × 1%) = 150% + 40% = 190%

### 防御减伤公式

`damage_reduction = defense / (defense + 100)`

**变量：**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| defense | DEF | int | 0-149 | 防御属性值（CON或INT） |

**输出范围：** 0.0 到 0.60（0%到60%减伤）
**示例：** 防御50 / (50 + 100) = 50 / 150 = 0.33（33%减伤）

### 最终伤害公式

`final_damage = (base_damage + attribute_bonus) × element_modifier × crit_multiplier × (1 - damage_reduction)`

**变量：**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_damage | D_b | int | 1-20 | 卡牌基础伤害 |
| attribute_bonus | A_b | float | 0-75 | 属性加成伤害 |
| element_modifier | M_e | float | 0.8-1.2 | 元素克制倍率 |
| crit_multiplier | M_c | float | 1.0-3.0 | 暴击倍率（非暴击为1.0） |
| damage_reduction | R_d | float | 0.0-0.6 | 防御减伤率 |

**输出范围：** 0（最低）到 150（最高，理论极限）
**示例：** (5 + 15) × 1.2 × 1.9 × (1 - 0.33) = 20 × 1.2 × 1.9 × 0.67 = 30.6 → 取整为31

## Edge Cases

1. **伤害值为0时**：如果计算结果≤0，伤害值设为1（最小伤害），避免出现负数或0伤害的尴尬情况。
2. **暴击率为0%时**：如果暴击率计算结果≤0，强制设为5%（基础值），避免永远不暴击。
3. **暴击率为100%时**：如果暴击率≥100%，强制设为50%（上限），避免100%暴击破坏平衡。
4. **元素属性为空时**：如果卡牌或目标无元素属性，元素克制倍率设为1.0（无克制）。
5. **防御值为0时**：如果防御值为0，减伤率为0%，直接应用全额伤害。
6. **多个减伤效果叠加时**：采用乘法叠加（如两个30%减伤 = 1-0.7×0.7 = 51%），避免100%减伤。
7. **属性值溢出时**：如果属性值超过149（上限），按149计算，避免数值溢出。
8. **卡牌效果值为0时**：如果卡牌效果值为0，伤害为0，但仍然触发其他效果（如状态效果）。

## Dependencies

### 硬依赖（系统无法正常运行）
- **角色属性系统**：获取攻击者/防御者的基础属性值（STR、INT、DEX、CON、PER、LCK）
- **卡牌数据库**：获取卡牌效果值、效果类型、元素属性

### 软依赖（增强功能）
- **状态效果系统**：获取Buff/Debuff对伤害的修正（如攻击力提升、防御力降低）

### 被依赖（其他系统依赖本系统）
- **召唤物系统**：调用伤害计算公式处理召唤物的攻击伤害
- **卡牌战斗系统**：调用伤害计算公式处理卡牌的直接伤害效果

## Tuning Knobs

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| **base_damage_multiplier** | 1.0 | 0.5-2.0 | 调整整体伤害水平，影响所有伤害计算 |
| **crit_base_rate** | 5% | 0-20% | 基础暴击概率，无属性加成时的暴击率 |
| **crit_per_coefficient** | 0.005 | 0.001-0.02 | PER属性对暴击率的影响系数 |
| **crit_damage_multiplier** | 1.5 | 1.0-3.0 | 暴击时的伤害倍率 |
| **element_advantage_multiplier** | 1.5 | 1.2-2.0 | 元素克制时的伤害倍率 |
| **element_disadvantage_multiplier** | 0.75 | 0.5-0.9 | 元素被克时的伤害倍率 |
| **defense_reduction_coefficient** | 0.01 | 0.005-0.02 | CON属性对减伤的影响系数 |
| **damage_reduction_stacking** | 乘法 | 乘法/加法 | 多个减伤效果的叠加方式 |

## Visual/Audio Requirements

### 视觉效果

| 事件 | 视觉效果 | 说明 |
|------|----------|------|
| 普通伤害 | 数字弹出（白色） | 显示伤害数值，向上漂浮消失 |
| 暴击伤害 | 数字弹出（金色）+ 放大效果 | 暴击时数字更大、更闪亮 |
| 元素克制 | 元素特效（火焰/冰霜/雷电） | 元素克制时显示对应元素特效 |
| 减伤效果 | 护盾闪烁 | 防御减伤时显示护盾效果 |

### 音频效果

| 事件 | 音效 | 说明 |
|------|------|------|
| 普通伤害 | 撞击音效 | 普通攻击命中时的音效 |
| 暴击伤害 | 强化撞击音效 + 暴击音效 | 暴击时的特殊音效 |
| 元素克制 | 元素音效（火焰/冰霜/雷电） | 元素克制时的对应音效 |
| 减伤效果 | 护盾音效 | 防御减伤时的音效 |

## UI Requirements

### 伤害数字弹出
- **位置**：目标上方
- **动画**：向上漂浮，1秒后消失
- **字体大小**：普通伤害16px，暴击伤害24px
- **颜色**：普通伤害白色，暴击伤害金色

### 状态提示
- **暴击提示**：显示"暴击！"文字，金色
- **元素克制提示**：显示"克制！"文字，对应元素颜色
- **减伤提示**：显示"减伤"文字，蓝色

### 布局
- 伤害数字和状态提示应避免重叠
- 多个伤害同时发生时，错开显示位置

## Acceptance Criteria

|------|----------|----------|
