# World Exploration System

> **Status**: Designed
> **Author**: User + Agents
> **Last Updated**: 2026-06-03
> **Implements Pillar**: Pillar 3 (Explore = Reward), Pillar 4 (Persistent Growth)

## Overview

世界探索系统是《命运卡牌局》的探索管理层，负责管理玩家在多个世界之间的探索和区域解锁。系统连接世界状态系统和区域系统，为玩家提供完整的探索体验。

## Player Fantasy

**"我在穿越不同的世界，每个世界都有独特的风景和挑战"**

玩家应该感受到：
- **探索欲**：每个世界都有新的发现
- **成就感**：解锁新世界是对我的奖励
- **沉浸感**：每个世界有独特的视觉和音效
- **策略性**：选择探索哪个世界
- **期待感**：下一个世界会有什么在等着我

**参考体验**：塞尔达传说的海拉鲁探索 + 空洞骑士的地图系统 + 杀戮尖塔的路径选择

## Detailed Design

### Core Rules

**世界结构**：
- 游戏有多个世界（森林、城堡、废墟、虚空）
- 每个世界有3-5个区域
- 世界之间有连接关系（线性/分支/隐藏）

**探索规则**：
- 玩家从第一个世界开始
- 解锁条件满足后可进入新世界
- 每个世界有独立的进度和完成度
- 世界之间可自由切换（已解锁的世界）

**探索内容**：
- **战斗节点**：与敌人战斗
- **事件节点**：触发剧情事件
- **宝箱节点**：获得奖励
- **NPC节点**：与NPC对话
- **隐藏节点**：需要特殊条件发现
- **Boss节点**：世界Boss战斗

### States and Transitions

| 当前状态 | 触发条件 | 目标状态 | 说明 |
|----------|----------|----------|------|
| 未解锁 | 满足解锁条件 | 已解锁 | 解锁新世界 |
| 已解锁 | 玩家进入 | 探索中 | 开始探索 |
| 探索中 | 探索完成 | 已完成 | 完成世界探索 |
| 已完成 | 重新进入 | 探索中 | 允许重复探索 |
| 隐藏 | 发现条件满足 | 已解锁 | 发现隐藏世界 |

### Interactions with Other Systems

| 交互系统 | 方向 | 接口 | 说明 |
|----------|------|------|------|
| 世界状态系统 | 输入 | get_world_state() | 获取世界状态 |
| 区域系统 | 输入 | get_area_data() | 获取区域数据 |
| 卡牌战斗系统 | 输出 | start_battle() | 触发战斗事件 |
| 对话系统 | 输出 | start_dialogue() | 触发对话事件 |
| NPC系统 | 输出 | interact_npc() | 触发NPC交互 |
| 存档系统 | 输出 | save_exploration_state() | 保存探索状态 |
| UI系统 | 输出 | update_world_map() | 更新世界地图 |

## Formulas

**世界解锁公式**：

`world_unlocked = (previous_world_completed) AND (choice_met) AND (mark_met)`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| previous_world_completed | - | bool | true/false | 前置世界是否完成 |
| choice_met | - | bool | true/false | 是否满足选择要求 |
| mark_met | - | bool | true/false | 是否满足印记要求 |

**输出范围**：true/false
**示例**：完成森林世界 AND 选择帮助村民 AND 善良印记≥3 → 解锁城堡世界

**探索完成度公式**：

`exploration_completion = (explored_nodes / total_nodes) × 100%`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| explored_nodes | - | int | 0-50 | 已探索节点数 |
| total_nodes | - | int | 10-50 | 总节点数 |

**输出范围**：0%到100%
**示例**：世界有30个节点，已探索18个 → completion=60%

**隐藏世界发现公式**：

`hidden_world_discovered = (all_marks_met) AND (all_choices_met) AND (discovery_chance_roll)`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| all_marks_met | - | bool | true/false | 是否满足所有印记要求 |
| all_choices_met | - | bool | true/false | 是否满足所有选择要求 |
| discovery_chance_roll | - | bool | true/false | 发现概率判定 |

**输出范围**：true/false
**示例**：满足所有印记和选择要求 AND 概率判定通过 → 发现隐藏世界

## Edge Cases

| # | 边界情况 | 处理方式 |
|---|----------|----------|
| 1 | **所有世界已完成** | 显示"世界完成"，解锁隐藏内容 |
| 2 | **世界解锁条件冲突** | 优先满足最近的条件 |
| 3 | **隐藏世界已发现** | 标记为已发现，不再隐藏 |
| 4 | **世界状态异常** | 验证状态合法性，修复异常 |
| 5 | **世界重复进入** | 允许重复进入，保留探索进度 |
| 6 | **世界连接断裂** | 验证连接完整性，修复断裂 |
| 7 | **世界进度丢失** | 从存档恢复状态 |
| 8 | **世界Boss未解锁** | 显示解锁条件，阻止进入 |

## Tuning Knobs

| 调优参数 | 默认值 | 安全范围 | 极端行为 |
|----------|--------|----------|----------|
| 世界数量 | 4 | 3-6 | 太少=内容不足，太多=过于复杂 |
| 区域数量/世界 | 4 | 3-5 | 太少=内容不足，太多=过于复杂 |
| 隐藏世界比例 | 25% | 10-30% | 太少=无惊喜，太多=过于分散 |
| 世界完成奖励 | 500金币 | 200-1000 | 太低=无动力，太高=破坏平衡 |
| 区域探索奖励 | 100金币 | 50-200 | 太低=无动力，太高=破坏平衡 |
| 隐藏发现概率 | 10% | 5-20% | 太低=不可发现，太高=无惊喜 |
| 世界重置冷却 | 24小时 | 1-72小时 | 太短=无意义，太长=过于限制 |
| 快速旅行解锁 | 完成世界 | 完成区域/完成世界 | 影响探索便利性 |

## Visual/Audio Requirements

**视觉效果**：
- **世界地图**：手绘风格的世界地图
- **世界切换**：世界切换动画
- **区域解锁**：区域图标发光 + 解锁动画
- **隐藏发现**：特殊光效 + 发现动画
- **世界完成**：完成动画 + 奖励显示
- **快速旅行**：传送动画

**音频效果**：
- **世界进入**：世界环境音效
- **区域解锁**：解锁音效
- **隐藏发现**：发现音效 + 惊喜音乐
- **世界完成**：完成音效 + 成就音乐
- **快速旅行**：传送音效

## UI Requirements

**世界探索界面**：
```
┌─────────────────────────────────────────┐
│  世界选择（世界标签）                   │
├─────────────────────────────────────────┤
│                                         │
│  世界地图（区域图标 + 连接线）          │
│                                         │
├─────────────────────────────────────────┤
│  区域信息（名称/状态/解锁条件）         │
├─────────────────────────────────────────┤
│  世界完成度（进度条 + 百分比）          │
├─────────────────────────────────────────┤
│  操作按钮（快速旅行/返回）              │
└─────────────────────────────────────────┘
```

**交互规则**：
- 点击世界标签 → 切换世界
- 点击区域图标 → 进入区域
- 点击快速旅行 → 传送到已探索区域
- 拖拽地图 → 移动视图
