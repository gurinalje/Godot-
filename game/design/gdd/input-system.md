# 输入系统

> **Status**: In Design  
> **Author**: [user + agents]  
> **Last Updated**: 2026-06-03  
> **Implements Pillar**: 支柱2 — 爽快的策略深度

## Overview

**输入系统**是游戏的基础设施层，负责处理所有玩家输入并将其转换为游戏可识别的操作指令。

**技术层面**：系统使用 Godot 的 InputMap 机制，将物理输入（键盘按键、鼠标点击、手柄按钮）映射为语义化的动作名称（如 `play_card`、`select_target`、`open_menu`）。支持键位重映射，允许玩家自定义操作。

**玩家影响**：虽然玩家不直接与输入系统交互，但它决定了卡牌选择的精准度、Combo操作的流畅度、以及UI导航的便捷性。输入延迟或误识别会直接破坏"爽快的策略深度"支柱。

**核心职责**：
- 将物理输入映射为游戏动作
- 支持键盘/鼠标和手柄双模式
- 提供输入缓冲机制（支持Combo预输入）
- 管理输入上下文（战斗/探索/菜单不同输入响应）

## Player Fantasy

> 玩家应感受到**"我的操作被精准执行"**——每一次卡牌选择、每一个Combo预输入、每一次目标指定，都感觉像是手和屏幕之间没有延迟。像弹钢琴一样，手指的意图直接转化为游戏动作。

**支柱对齐**：支柱2（爽快的策略深度）— 精准的输入是策略深度的前提

## Detailed Design

### Core Rules

**输入映射架构**：

| 动作名称 | 键盘默认 | 鼠标默认 | 手柄默认 | 上下文 |
|----------|----------|----------|----------|--------|
| `play_card` | Space | 左键点击 | A按钮 | 战斗 |
| `select_card` | 方向键 | 鼠标移动 | 左摇杆 | 战斗 |
| `select_target` | Tab | 右键点击 | X按钮 | 战斗 |
| `end_turn` | Enter | — | Y按钮 | 战斗 |
| `open_deck` | D | — | LB | 探索/战斗 |
| `open_menu` | Esc | — | Start | 全局 |
| `confirm` | Enter | 左键点击 | A按钮 | 菜单/对话 |
| `cancel` | Esc | 右键点击 | B按钮 | 菜单/对话 |
| `move_up/down/left/right` | W/S/A/D | — | 左摇杆 | 探索 |

**输入缓冲机制**：
- 缓冲窗口：100ms（默认）
- 允许在动画播放期间预输入下一个动作
- 缓冲队列：最多存储2个预输入动作
- 优先级：最新输入覆盖旧输入

**输入上下文管理**：
- 使用状态机管理输入上下文
- 上下文切换时，禁用不相关的输入动作
- 上下文优先级：菜单 > 对话 > 战斗 > 探索

### States and Transitions

**输入上下文状态机**：

```
┌─────────────────────────────────────────────────────────┐
│                    全局输入层                            │
│  open_menu (Esc/Start) → 暂停游戏，打开菜单             │
└─────────────────────┬───────────────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────────────┐
│                    探索上下文                            │
│  move_* (WASD/左摇杆) → 角色移动                        │
│  confirm (Enter/A) → 与NPC/物品交互                     │
│  open_deck (D/LB) → 打开卡组界面                        │
│  进入战斗区域 → 切换到战斗上下文                        │
└─────────────────────┬───────────────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────────────┐
│                    战斗上下文                            │
│  select_card (方向键/摇杆) → 选择手牌                    │
│  play_card (Space/A) → 打出选中的卡牌                   │
│  select_target (Tab/X) → 选择目标                       │
│  end_turn (Enter/Y) → 结束回合                          │
│  open_deck (D/LB) → 查看卡组                            │
│  战斗结束 → 切换回探索上下文                            │
└─────────────────────┬───────────────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────────────┐
│                    对话上下文                            │
│  confirm (Enter/A) → 继续对话/选择选项                   │
│  cancel (Esc/B) → 跳过对话                              │
│  方向键/摇杆 → 选择对话选项                             │
│  对话结束 → 切换回探索上下文                            │
└─────────────────────┬───────────────────────────────────┘
                      │
┌─────────────────────▼───────────────────────────────────┐
│                    菜单上下文                            │
│  confirm (Enter/A) → 确认选择                           │
│  cancel (Esc/B) → 返回/取消                             │
│  方向键/摇杆 → 导航菜单                                 │
│  菜单关闭 → 切换回之前的上下文                          │
└─────────────────────────────────────────────────────────┘
```

**上下文切换规则**：
- 高优先级上下文可中断低优先级上下文
- 低优先级上下文不能中断高优先级上下文
- 上下文切换时，清空输入缓冲队列

### Interactions with Other Systems

| 系统 | 数据流向 | 接口描述 |
|------|----------|----------|
| **UI系统** | 输出 | 提供输入动作事件，UI系统响应导航和选择 |
| **卡牌战斗系统** | 输出 | 提供卡牌选择、打出、目标指定等战斗输入事件 |
| **对话系统** | 输出 | 提供对话继续、选项选择等对话输入事件 |
| **世界探索系统** | 输出 | 提供角色移动、交互等探索输入事件 |
| **规则改写系统** | 输入 | 接收规则变更，动态修改输入映射（如"所有卡牌自动选择目标"） |
| **存档系统** | 输出 | 提供键位配置数据，支持保存/加载自定义键位 |

**数据格式约定**：
- 输入系统发出信号：`input_action(action_name: String, context: String)`
- 规则改写系统可注册输入修改器：`register_input_modifier(modifier: InputModifier)`
- 存档系统通过 `save_keybindings() -> Dictionary` 和 `load_keybindings(data: Dictionary)` 接口

## Formulas

**输入缓冲公式**：

`buffer_valid = (current_time - input_time) <= BUFFER_WINDOW`

**变量**：
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| current_time | t | float | 0–∞ | 当前游戏时间（毫秒） |
| input_time | t_i | float | 0–∞ | 输入事件发生时间（毫秒） |
| BUFFER_WINDOW | W | int | 50–200 | 缓冲窗口大小（毫秒） |

**输出范围**：true/false — 输入是否在缓冲窗口内

**示例**：W=100ms，输入在t=5000ms，当前t=5050ms → buffer_valid = (5050-5000) <= 100 = true

---

**输入延迟计算**：

`total_latency = hardware_latency + engine_latency + processing_latency`

**变量**：
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| hardware_latency | L_h | float | 1–20 | 硬件输入延迟（毫秒） |
| engine_latency | L_e | float | 0–5 | Godot引擎处理延迟（毫秒） |
| processing_latency | L_p | float | 0–3 | 输入系统处理延迟（毫秒） |

**输出范围**：1–28ms — 目标总延迟 < 16ms（一帧）

## Edge Cases

- **如果同时按下多个冲突按键**：按优先级处理（最新输入覆盖旧输入，或按动作优先级）
- **如果输入缓冲队列已满**：丢弃最早的输入，保留最新的2个
- **如果上下文切换时有未处理的输入**：清空缓冲队列，防止误操作
- **如果手柄断开连接**：自动切换到键盘/鼠标模式，显示提示
- **如果键位映射冲突**：提示玩家冲突，允许选择保留哪个映射
- **如果输入延迟超过阈值**：记录性能警告，但不阻断游戏
- **如果规则改写系统修改了输入映射**：立即生效，无需重启

## Dependencies

**硬依赖（系统无法运行）**：无

**软依赖（增强但可独立运行）**：
| 系统 | 依赖类型 | 描述 |
|------|----------|------|
| **规则改写系统** | 软依赖 | 提供动态输入映射修改能力 |
| **存档系统** | 软依赖 | 提供键位配置持久化 |

**被依赖（其他系统依赖本系统）**：
| 系统 | 依赖类型 | 描述 |
|------|----------|------|
| **UI系统** | 硬依赖 | 需要输入事件来驱动UI交互 |
| **卡牌战斗系统** | 硬依赖 | 需要战斗输入事件 |
| **对话系统** | 硬依赖 | 需要对话输入事件 |
| **世界探索系统** | 硬依赖 | 需要移动和交互输入事件 |

## Tuning Knobs

| 参数 | 默认值 | 范围 | 描述 | 影响 |
|------|--------|------|------|------|
| `BUFFER_WINDOW` | 100 | 50–200 | 输入缓冲窗口（毫秒） | 越大越宽松，但可能感觉迟钝 |
| `MAX_BUFFER_QUEUE` | 2 | 1–5 | 最大缓冲队列长度 | 越大Combo越容易，但可能误操作 |
| `CONTEXT_SWITCH_CLEAR` | true | true/false | 上下文切换时清空缓冲 | false可能导致误操作 |
| `GAMEPAD_DEADZONE` | 0.2 | 0.1–0.5 | 手柄摇杆死区 | 越大越不敏感 |
| `MOUSE_SENSITIVITY` | 1.0 | 0.5–2.0 | 鼠标灵敏度 | 影响卡牌选择精度 |
| `KEY_REPEAT_DELAY` | 500 | 300–1000 | 按键重复延迟（毫秒） | 影响方向键导航速度 |
| `KEY_REPEAT_RATE` | 50 | 30–100 | 按键重复速率（毫秒） | 影响方向键连续导航速度 |
| `INPUT_LATENCY_THRESHOLD` | 16 | 8–32 | 输入延迟警告阈值（毫秒） | 超过此值记录性能警告 |

## Visual/Audio Requirements

**视觉反馈**：
- 输入缓冲激活时，显示短暂的视觉提示（如卡牌边缘闪光）
- 上下文切换时，UI元素有过渡动画
- 手柄断开连接时，显示警告图标

**音频反馈**：
- 输入缓冲激活时，播放轻微的"咔嗒"音效
- 上下文切换时，播放过渡音效
- 键位冲突时，播放警告音效

## UI Requirements

**键位设置界面**：
- 显示所有可自定义的动作和当前绑定
- 支持键盘/鼠标和手柄两套独立配置
- 提供"重置为默认"按钮
- 键位冲突时显示警告

**输入提示**：
- 根据当前输入设备显示对应的按键提示
- 支持动态切换（检测到手柄输入时切换提示）
- 在战斗UI中显示当前可用操作的快捷键

## Acceptance Criteria

| 编号 | 验收标准 | 测试方法 |
|------|----------|----------|
| AC-1 | 所有默认键位映射正确工作 | 逐一测试每个动作的默认键位 |
| AC-2 | 玩家可自定义键位映射 | 修改键位后验证功能正常 |
| AC-3 | 键位冲突时显示警告 | 尝试映射已使用的键位 |
| AC-4 | 输入缓冲在100ms内正常工作 | 快速连续输入验证缓冲 |
| AC-5 | 上下文切换正确禁用/启用输入 | 在不同上下文测试输入响应 |
| AC-6 | 手柄输入与键盘/鼠标一致 | 使用手柄测试所有动作 |
| AC-7 | 手柄断开时自动切换输入模式 | 拔掉手柄验证切换 |
| AC-8 | 输入延迟低于16ms | 使用性能分析工具测量 |
| AC-9 | 规则改写可动态修改输入映射 | 激活规则后验证输入变化 |
| AC-10 | 键位配置正确保存和加载 | 保存后重启游戏验证配置 |

## Open Questions

1. **输入录制与回放**：是否需要支持输入录制功能（用于测试或回放精彩操作）？
2. **宏命令**：是否允许玩家设置宏命令（一键执行多个操作）？
3. **触屏支持**：未来是否考虑触屏设备支持？
4. **无障碍输入**：是否需要支持单手操作或其他无障碍输入方案？
5. **输入分析**：是否需要收集输入数据用于分析玩家行为？
