# 规则改写系统 (Rule Rewriting System)

> **Status**: In Design  
> **Author**: [user + agents]  
> **Last Updated**: 2026-06-03  
> **Implements Pillar**: 支柱1 — 选择即规则

## Overview

**规则改写系统**是游戏的核心创新系统，负责管理玩家选择对战斗规则的永久改变。它定义了规则改写的类型、触发条件、效果和持续时间，为卡牌战斗系统提供规则改写支持。玩家通过规则改写系统感受到"我的选择真的改变了游戏的运作方式"——每一个决定都可能改变战斗规则，使游戏玩法更加多样化。

规则改写系统读取选择系统的选择结果，结合故事印记系统的印记效果，实现规则的永久改变。玩家通过规则改写系统感受到"我的选择产生了有意义的结果，游戏规则因为我之前的选择而变得不同"。

## Player Fantasy

**规则改写系统**是玩家直接体验的系统——玩家通过做出选择来改变游戏规则，规则的改变会影响战斗体验。

**情感目标**："我的选择真的改变了游戏的运作方式"

**玩家应该感受到**：
- **掌控感**：我的选择直接影响游戏规则
- **策略深度**：我需要考虑选择的长期影响，而不仅仅是眼前利益
- **惊喜感**：发现规则改变如何影响战斗时的惊喜

**情感锚点**：像杀戮尖塔中遗物改变游戏规则的爽快感——"我获得了这个遗物，现在所有攻击卡牌费用-1"

## Detailed Design

### Core Rules

1. **规则改写类型**：
   - **卡牌效果改变**：改变卡牌的效果（如伤害+20%、费用-1）
   - **战斗规则改变**：改变战斗规则（如每回合多抽1张牌、能量上限+1）
   - **召唤物规则改变**：改变召唤物规则（如召唤物场上上限+1、召唤物生命值+20%）
   - **环境规则改变**：改变环境规则（如环境效果持续时间+1回合）

2. **规则改写触发条件**：
   - **选择触发**：做出特定选择时触发规则改写
   - **印记触发**：获得特定印记时触发规则改写
   - **卡牌触发**：使用特定卡牌时触发规则改写
   - **战斗触发**：在特定战斗条件下触发规则改写

3. **规则改写效果**：
   - **永久效果**：规则改写永久生效，直到游戏结束
   - **临时效果**：规则改写持续一定回合数
   - **条件效果**：规则改写在特定条件下生效

4. **规则改写限制**：
   - **规则改写上限**：最多同时存在5个规则改写
   - **规则改写冲突**：同类规则改写取最高值
   - **规则改写优先级**：规则改写按优先级顺序应用

### States and Transitions

| 状态 | 描述 | 触发条件 |
|------|------|----------|
| **未激活** | 规则改写已解锁但未激活 | 满足激活条件时进入 |
| **激活中** | 规则改写正在生效 | 激活条件满足时进入 |
| **暂停中** | 规则改写暂时失效 | 暂停条件满足时进入 |
| **结束** | 规则改写已结束 | 持续时间结束时进入 |

**状态转换规则**：
- 未激活 → 激活中：满足激活条件时自动进入
- 激活中 → 暂停中：满足暂停条件时自动进入
- 暂停中 → 激活中：暂停条件解除时自动进入
- 激活中 → 结束：持续时间结束时自动进入

### Interactions with Other Systems

| 系统 | 数据流向 | 接口描述 |
|------|----------|----------|
| **选择系统** | 输入 | 读取选择结果，触发规则改写 |
| **故事印记系统** | 输入 | 读取印记效果，触发规则改写 |
| **卡牌战斗系统** | 输出 | 提供规则改写效果，改变战斗规则 |
| **UI系统** | 输出 | 提供规则改写显示、激活提示 |
| **存档系统** | 输出 | 保存规则改写状态 |

## Formulas

### 规则改写效果公式

`rule_effect = base_value × (1 + effect_bonus) × mark_multiplier`

**变量：**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_value | V_b | float | 0.1-2.0 | 规则改写基础效果值 |
| effect_bonus | E_bonus | float | 0-1.0 | 效果加成（来自印记/选择） |
| mark_multiplier | M_m | float | 0.5-2.0 | 印记倍率 |
| rule_effect | R_e | float | 0.1-4.0 | 规则改写最终效果值 |

**输出范围：** 0.1到4.0（正常游戏范围）
**示例：** 基础效果值0.2（+20%），加成10%，印记倍率1.2x，效果 = 0.2 × 1.1 × 1.2 = 0.264（+26.4%）

### 规则改写持续时间公式

`rule_duration = base_duration × (1 + duration_bonus)`

**变量：**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_duration | D_b | int | 1-10 | 规则改写基础持续时间（回合） |
| duration_bonus | D_bonus | float | 0-1.0 | 持续时间加成 |
| rule_duration | D_r | int | 1-20 | 规则改写最终持续时间（回合） |

**输出范围：** 1到20（正常游戏范围）
**示例：** 基础持续时间3回合，加成20%，持续时间 = 3 × 1.2 = 3.6 ≈ 4回合

### 规则改写优先级公式

`rule_priority = base_priority + mark_bonus + choice_bonus`

**变量：**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_priority | P_b | int | 1-10 | 规则改写基础优先级 |
| mark_bonus | P_m | int | 0-5 | 印记加成 |
| choice_bonus | P_c | int | 0-5 | 选择加成 |
| rule_priority | P_r | int | 1-20 | 规则改写最终优先级 |

**输出范围：** 1到20（正常游戏范围）
**示例：** 基础优先级5，印记加成2，选择加成3，优先级 = 5 + 2 + 3 = 10

## Edge Cases

- **如果规则改写数量达到上限（5个）**：无法激活新规则改写，显示"规则改写已满"提示。
- **如果同类规则改写冲突**：取最高值，显示"规则改写冲突"提示。
- **如果规则改写效果为负**：规则改写效果为负时，显示"规则改写负面效果"提示。
- **如果规则改写持续时间结束**：规则改写自动结束，显示"规则改写已结束"提示。
- **如果规则改写在战斗中激活**：规则改写立即生效，显示"规则改写激活"提示。
- **如果规则改写在对话中激活**：规则改写在下一次战斗中生效。
- **如果规则改写在探索中激活**：规则改写立即生效，显示"规则改写激活"提示。
- **如果存档丢失**：规则改写状态丢失，需要重新触发。

## Dependencies

### 硬依赖（系统无法正常工作）

| 系统 | 依赖原因 |
|------|----------|
| **选择系统** | 规则改写需要选择系统触发 |

### 软依赖（系统可以工作但功能受限）

| 系统 | 依赖原因 |
|------|----------|
| **故事印记系统** | 规则改写可以受故事印记影响 |
| **对话系统** | 规则改写可以在对话中触发 |

### 被依赖（其他系统依赖本系统）

| 系统 | 依赖原因 |
|------|----------|
| **卡牌战斗系统** | 卡牌战斗系统需要读取规则改写效果 |

## Tuning Knobs

| 调优参数 | 默认值 | 范围 | 说明 |
|----------|--------|------|------|
| **规则改写数量上限** | 5 | 1-10 | 同时激活的规则改写数量 |
| **规则改写效果加成** | 1.2x | 1.1x-2.0x | 规则改写效果加成倍率 |
| **规则改写持续时间** | 3回合 | 1-10回合 | 规则改写持续时间 |
| **规则改写触发概率** | 50% | 10%-100% | 规则改写触发概率 |
| **规则改写冲突抵消比例** | 50% | 10%-100% | 同类规则改写冲突时的效果抵消比例 |
| **规则改写解锁条件** | 10 | 1-20 | 解锁规则改写所需的选择次数 |
| **规则改写印记系数** | 0.5 | 0.1-1.0 | 故事印记对规则改写的影响系数 |
| **规则改写优先级** | 1 | 1-5 | 规则改写优先级（数值越大优先级越高） |

## Visual/Audio Requirements

### 视觉效果
- **规则改写激活动画**：规则改写激活时，规则改写图标从屏幕中央浮现，带有金色光晕
- **规则改写效果动画**：规则改写效果应用时，效果区域闪烁
- **规则改写持续时间显示**：规则改写持续时间显示在规则改写图标下方
- **规则改写冲突提示**：规则改写冲突时，显示"规则改写冲突"提示
- **规则改写解锁动画**：规则改写解锁时，规则改写图标解锁，带有金色光晕

### 音频效果
- **规则改写激活音效**：规则改写激活时，播放激活音效（像打开一本新书的声音）
- **规则改写效果音效**：规则改写效果应用时，播放效果音效（像魔法生效的声音）
- **规则改写持续时间音效**：规则改写持续时间结束时，播放结束音效（像魔法消散的声音）
- **规则改写冲突音效**：规则改写冲突时，播放冲突音效（像金属碰撞的声音）
- **规则改写解锁音效**：规则改写解锁时，播放解锁音效（像解锁成就的声音）

## UI Requirements

### 规则改写状态显示
- **位置**：战斗界面左上角，玩家信息区域
- **内容**：当前激活的规则改写图标、名称、效果
- **大小**：每个规则改写图标30x30像素
- **动画**：规则改写激活时，图标从屏幕中央飞入规则改写区域

### 规则改写详情界面
- **位置**：点击规则改写图标后弹出
- **内容**：规则改写名称、效果、持续时间、触发条件
- **大小**：300x200像素
- **交互**：点击空白处关闭

### 规则改写冲突提示
- **位置**：屏幕中央
- **内容**："规则改写冲突"提示
- **大小**：200x100像素
- **动画**：提示从屏幕中央浮现，然后消失

### 规则改写解锁提示
- **位置**：屏幕中央
- **内容**："规则改写已解锁"提示
- **大小**：200x100像素
- **动画**：提示从屏幕中央浮现，然后消失
