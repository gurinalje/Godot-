# NPC System

> **Status**: In Design
> **Author**: User + Agents
> **Last Updated**: 2026-06-03
> **Implements Pillar**: Pillar 1 (Choice = Rules), Pillar 3 (Explore = Reward)

## Overview

NPC系统是《命运卡牌局》的角色管理层，负责定义和管理游戏中的非玩家角色（NPC）。系统管理NPC的对话、关系、态度和行为，为玩家提供丰富的交互体验。

## Player Fantasy

**"每个NPC都有自己的故事和性格"**

玩家应该感受到：
- **沉浸感**：NPC有自己的性格和故事
- **关系感**：我的选择影响NPC对我的态度
- **惊喜感**：NPC会根据我的选择改变行为
- **策略性**：与NPC的关系影响游戏进程
- **情感连接**：我会对某些NPC产生情感

**参考体验**：巫师3的NPC互动 + 星露谷物语的关系系统 + 传说之下的角色互动

## Detailed Design

### Core Rules

**NPC类型**：
- **商人NPC**：出售卡牌和道具
- **任务NPC**：提供任务和奖励
- **剧情NPC**：推进主线剧情
- **隐藏NPC**：需要特殊条件发现
- **中立NPC**：可攻击或帮助

**NPC属性**：
- **名称**：NPC的显示名称
- **类型**：NPC的功能类型
- **态度**：对玩家的态度（-100到+100）
- **状态**：NPC的当前状态（正常/受伤/死亡/离开）
- **位置**：NPC所在的区域和节点
- **对话树**：NPC的对话内容和选择
- **库存**：商人NPC的商品列表
- **任务**：任务NPC的任务列表

**NPC交互规则**：
- 玩家只能与同一节点的NPC交互
- NPC态度影响对话选项和价格
- NPC状态影响可用交互
- NPC死亡后相关剧情终止

### States and Transitions

| 当前状态 | 触发条件 | 目标状态 | 说明 |
|----------|----------|----------|------|
| 正常 | 玩家交互 | 交互中 | 开始对话/交易 |
| 交互中 | 交互完成 | 正常 | 结束交互 |
| 正常 | 受到伤害 | 受伤 | NPC受伤 |
| 受伤 | 恢复治疗 | 正常 | NPC恢复 |
| 正常 | 被攻击致死 | 死亡 | NPC死亡 |
| 死亡 | 无法恢复 | 死亡 | 永久死亡 |
| 正常 | 任务完成 | 离开 | NPC离开区域 |
| 离开 | 重新出现 | 正常 | NPC重新出现 |

## Formulas

**NPC态度公式**：

`npc_attitude = base_attitude + choice_modifier + mark_modifier + reputation_modifier`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| base_attitude | - | int | -50~+50 | NPC基础态度 |
| choice_modifier | - | int | -30~+30 | 选择影响 |
| mark_modifier | - | int | -20~+20 | 印记影响 |
| reputation_modifier | - | int | -10~+10 | 声望影响 |

**输出范围**：-100到+100
**示例**：NPC基础态度+20，选择影响-10，印记影响+5，声望影响+3 → attitude=+18

**NPC价格公式**：

`price = base_price × (1 - attitude / 200)`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| base_price | - | int | 10-1000 | 基础价格 |
| attitude | - | int | -100~+100 | NPC态度 |

**输出范围**：50%到150%基础价格
**示例**：基础价格100，态度+50 → price=100×0.75=75

**NPC任务奖励公式**：

`reward = base_reward × (1 + attitude / 100)`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| base_reward | - | int | 50-500 | 基础奖励 |
| attitude | - | int | -100~+100 | NPC态度 |

**输出范围**：0%到200%基础奖励
**示例**：基础奖励100，态度+30 → reward=100×1.3=130

## Edge Cases

| # | 边界情况 | 处理方式 |
|---|----------|----------|
| 1 | **NPC死亡** | 相关剧情终止，但印记保留 |
| 2 | **NPC态度极低** | NPC拒绝交互，显示敌对状态 |
| 3 | **NPC态度极高** | NPC提供额外奖励和隐藏内容 |
| 4 | **NPC库存为空** | 显示"无商品"，允许退出 |
| 5 | **NPC任务失败** | 允许重试，不消耗资源 |
| 6 | **NPC位置冲突** | 验证位置合法性，修复冲突 |
| 7 | **NPC状态异常** | 验证状态合法性，修复异常 |
| 8 | **NPC对话缺失** | 显示默认对话，允许退出 |

## Tuning Knobs

| 调优参数 | 默认值 | 安全范围 | 极端行为 |
|----------|--------|----------|----------|
| NPC数量/区域 | 5 | 3-10 | 太少=内容不足，太多=管理复杂 |
| 态度变化速度 | 1.0 | 0.5-2.0 | 太慢=无感，太快=过于敏感 |
| 价格折扣上限 | 50% | 20-70% | 太低=无奖励，太高=破坏平衡 |
| 任务奖励倍率 | 1.0 | 0.5-2.0 | 影响任务奖励 |
| NPC死亡惩罚 | 高 | 低/中/高 | 影响NPC死亡后果 |
| NPC复活时间 | 永久 | 1小时/1天/永久 | 影响NPC死亡后果 |
| 态度显示阈值 | ±20 | ±10~±50 | 影响态度可见性 |
| 隐藏NPC比例 | 10% | 5-20% | 太少=无惊喜，太多=过于分散 |

## Visual/Audio Requirements

**视觉效果**：
- **NPC立绘**：每个NPC有独特的立绘
- **态度指示**：NPC头顶显示态度图标
- **交互提示**：NPC附近显示交互提示
- **对话界面**：对话框显示NPC立绘和文字
- **交易界面**：商人NPC的交易界面
- **任务界面**：任务NPC的任务界面

**音频效果**：
- **NPC对话**：NPC有独特的语音（可选）
- **交互音效**：不同交互有不同音效
- **态度变化**：态度变化音效
- **交易完成**：交易完成音效
- **任务完成**：任务完成音效

## UI Requirements

**NPC交互界面**：
```
┌─────────────────────────────────────────┐
│  NPC名称 + 类型 + 态度                  │
├─────────────────────────────────────────┤
│                                         │
│  NPC立绘                                │
│                                         │
├─────────────────────────────────────────┤
│  对话框（NPC对话文字）                  │
├─────────────────────────────────────────┤
│  选择选项（对话选择）                   │
│  ┌─────┐  ┌─────┐  ┌─────┐             │
│  │  A  │  │  B  │  │  C  │             │
│  └─────┘  └─────┘  └─────┘             │
├─────────────────────────────────────────┤
│  交互按钮（对话/交易/任务/离开）        │
└─────────────────────────────────────────┘
```

**交互规则**：
- 点击NPC → 打开交互界面
- 点击对话选项 → 继续对话
- 点击交易按钮 → 打开交易界面
- 点击任务按钮 → 打开任务界面
- 点击离开按钮 → 关闭交互界面

## Acceptance Criteria

| # | 验收标准 | Given-When-Then |
|---|----------|-----------------|
| AC-1 | NPC加载 | GIVEN 玩家进入区域 WHEN 区域加载 THEN 所有NPC正确显示 |
| AC-2 | NPC交互 | GIVEN 玩家点击NPC WHEN 触发交互 THEN 交互界面正确显示 |
| AC-3 | NPC对话 | GIVEN 玩家与NPC对话 WHEN 对话进行 THEN 对话正确显示 |
| AC-4 | NPC交易 | GIVEN 玩家与商人交易 WHEN 交易进行 THEN 交易界面正确显示 |
| AC-5 | NPC任务 | GIVEN 玩家接受任务 WHEN 任务进行 THEN 任务界面正确显示 |
| AC-6 | 态度变化 | GIVEN 玩家做出选择 WHEN 态度变化 THEN 态度正确更新 |
| AC-7 | NPC状态 | GIVEN NPC状态变更 WHEN 状态变化 THEN 状态正确更新 |
| AC-8 | NPC死亡 | GIVEN NPC死亡 WHEN 死亡判定 THEN NPC状态正确更新 |
| AC-9 | 状态保存 | GIVEN NPC状态变更 WHEN 保存游戏 THEN 状态正确保存 |
| AC-10 | 状态加载 | GIVEN 加载存档 WHEN 游戏启动 THEN NPC状态正确恢复 |
