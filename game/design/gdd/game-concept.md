## 设计支柱

### 支柱1：选择即规则（Choice = Rules）

> 每个有意义的选择都应该改变游戏的运作方式，而不仅仅是改变对话文本。

**设计测试**：如果一个选择只改变NPC的台词，但不影响战斗/卡牌/规则，那它不够重要。

### 支柱2：爽快的策略深度（Satisfying Depth）

> 卡牌战斗应该既有深度（需要思考），又有爽感（Combo连锁、即时反馈）。

**设计测试**：如果一场战斗让玩家感到"无聊"或"太难"，都是失败的。

### 支柱3：探索奖励好奇心（Explore = Reward）

> 玩家的每一次探索都应该有回报——可能是卡牌、故事碎片、或隐藏规则。

**设计测试**：如果玩家探索了一个隐藏区域却什么都没发现，那是设计失误。

### 支柱4：持久的成长感（Persistent Growth）

> 玩家应该清楚地感受到自己在变强——卡组在壮大、角色在成长、世界在展开。

**设计测试**：如果玩家玩了1小时后感觉"和开始时一样"，那是失败的。

---

## 反支柱

| 反支柱 | 原因 |
|--------|------|
| **不是纯Roguelike** | 我们不做"每次从零开始"的设计。玩家的进度和卡组是持久的。破坏"持久成长感"支柱。 |
| **不是纯数值碾压** | 我们不做"等级够高就能赢"的设计。策略和选择比数值更重要。破坏"爽快的策略深度"支柱。 |
| **不是开放世界沙盒** | 我们不做"无限自由"的设计。探索是有目的的，每个区域都有明确的内容。破坏"探索奖励好奇心"支柱。 |

## 核心循环

### 30秒循环（战斗瞬间）

```
打出卡牌 → 触发连锁Combo → 华丽特效爆发 → 即时反馈 → 爽！
```

### 5分钟循环（局间体验）

```
探索区域 ↔ 对话NPC ↔ 管理卡组 ↔ 10-15分钟卡牌对战（可SL回档）
```

### Session循环（30-120分钟）

```
选择区域 → 探索发现 → 触发剧情 → 卡组调整 → 挑战关卡 → 解锁新区域
```

### 进度循环（长期）

```
做出选择 → 留下故事印记 → 解锁新世界 → 获得新卡牌池 → 改变规则 → 循环
```

## 卡牌系统

### 四大卡牌类型

| 类型 | 功能 | 设计意图 |
|------|------|----------|
| **召唤类** | 召唤生物/单位到场上 | 类似游戏王，创造场面优势 |
| **直接伤害类** | 对敌方造成即时伤害 | 炉石的核心伤害机制 |
| **环境阵地类** | 改变战场环境/地形 | 创造策略深度，影响召唤物 |
| **增益减益类** | 给己方/敌方添加Buff/Debuff | 策略组合的关键 |

### 选择如何影响卡牌

| 选择类型 | 影响方式 | 示例 |
|----------|----------|------|
| **元素切换** | 解锁不同类型的卡牌池 | 选择火元素→解锁火系卡牌 |
| **规则改写** | 永久改变战斗规则 | "所有召唤物获得飞行" |
| **故事印记** | 印记影响可用卡牌 | 选择帮助NPC→获得专属卡牌 |

## 世界与探索

### 世界结构

- **多世界系统**：每个世界有独特的视觉风格、卡牌池、故事线
- **区域解锁**：通过选择和进度解锁新区域
- **隐藏内容**：彩蛋剧情、隐藏Boss、稀有卡牌

### 探索回报

- **卡牌奖励**：新卡牌、稀有卡牌
- **故事碎片**：剧情片段、世界观信息
- **规则解锁**：新战斗规则、卡牌组合
- **角色成长**：属性提升、技能解锁

---

## 叙事系统

### 三层剧情结构

| 层级 | 类型 | 触发方式 |
|------|------|----------|
| **主线剧情** | 核心故事 | 推进游戏进度 |
| **连环剧情** | 分支故事 | 特定选择触发 |
| **隐藏剧情** | 彩蛋/秘密 | 特殊条件触发 |

### 选择影响机制

- **即时影响**：选择立即改变当前对话/事件
- **延迟影响**：选择在后续剧情中产生后果
- **规则影响**：选择改变可用卡牌和战斗规则
- **世界影响**：选择决定解锁哪个世界/区域

## 玩家动机

### 自我决定理论分析

| 动机 | 满足方式 |
|------|----------|
| **自主性** | 选择改变规则和故事，玩家感到"我的选择有意义" |
| **胜任感** | 深度卡牌组合需要掌握，SL降低门槛 |
| **关联性** | NPC关系、剧情选择影响角色互动 |

### 驱动力分析

| 驱动力 | 满足方式 |
|--------|----------|
| **收集欲** | 解锁所有卡牌、发现所有剧情、完成所有世界 |
| **挑战欲** | 打败困难Boss、完成高难度挑战 |
| **故事欲** | 看不同选择的结果、发现隐藏剧情 |

## MVP定义

### 最小可行产品包含

1. **1个完整世界**：有独特视觉风格、卡牌池、故事线
2. **核心卡牌战斗**：4种卡牌类型、连锁Combo、即时反馈
3. **选择系统**：至少3个有影响的选择点
4. **基础RPG元素**：角色成长、卡组管理
5. **1个完整故事线**：有开头、发展、结局

### MVP目标

验证"选择改变规则"这个核心假设是否有趣
