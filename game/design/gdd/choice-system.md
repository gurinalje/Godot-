# Choice System

> **Status**: In Design
> **Author**: User + Agents
> **Last Updated**: 2026-06-03
> **Implements Pillar**: Pillar 1 (Choice = Rules), Pillar 3 (Explore = Reward)

## Overview

选择系统是《命运卡牌局》的核心创新系统，负责管理玩家在游戏中的所有有意义的选择，以及这些选择如何改变游戏规则、故事走向和卡牌效果。系统记录玩家的每一个重要选择，通过故事印记、元素切换和规则改写三个机制，将选择的影响持久化并应用到后续的游戏体验中。

## Player Fantasy

**"我的选择真的改变了这个世界"**

玩家应该感受到：
- **掌控感**：我的每一个决定都在塑造游戏的走向
- **惊喜感**：选择带来的后果在后续剧情中出乎意料
- **策略深度**：选择不仅影响故事，还影响卡牌和战斗规则
- **重玩价值**：不同的选择会带来完全不同的游戏体验
- **责任感**：我的选择有重量，需要慎重考虑

**参考体验**：传说之下的选择后果 + 杀戮尖塔的卡牌构筑变化 + 巫师3的道德困境

## Detailed Design

### Core Rules

**选择类型**：
1. **即时选择**：对话中的A/B/C选项，立即产生效果
2. **延迟选择**：选择的效果在后续剧情中才显现
3. **规则选择**：选择改变战斗规则（通过规则改写系统）
4. **印记选择**：选择留下故事印记（通过故事印记系统）

**选择触发条件**：
- 对话分支：NPC对话中的选择点
- 剧情节点：主线/支线剧情的关键决策
- 探索发现：隐藏区域的互动选择
- 战斗决策：战斗中的特殊选择（如放过敌人）

**选择影响机制**：
- **即时影响**：选择立即改变当前对话/事件
- **延迟影响**：选择在后续剧情中产生后果
- **规则影响**：选择改变可用卡牌和战斗规则
- **世界影响**：选择决定解锁哪个世界/区域

**选择限制**：
- 每个选择点只能选择一次（不可撤销）
- 选择有时间限制（对话中的选择）
- 选择有前置条件（需要特定印记或卡牌）

### States and Transitions

| 当前状态 | 触发条件 | 目标状态 | 说明 |
|----------|----------|----------|------|
| 等待选择 | 到达选择点 | 选择中 | 显示选择界面 |
| 选择中 | 玩家做出选择 | 选择确认 | 记录选择 |
| 选择确认 | 选择生效 | 选择完成 | 应用效果 |
| 选择完成 | 效果应用完成 | 等待选择 | 准备下一个选择 |
| 选择锁定 | 选择已做出 | 不可更改 | 防止重复选择 |

### Interactions with Other Systems

| 交互系统 | 方向 | 接口 | 说明 |
|----------|------|------|------|
| 故事印记系统 | 输出 | add_mark(type, value) | 选择留下印记 |
| 元素系统 | 输出 | unlock_element(element) | 选择解锁元素 |
| 规则改写系统 | 输出 | rewrite_rule(rule, effect) | 选择改变规则 |
| 对话系统 | 输入 | get_choice_point() | 获取对话选择点 |

| UI系统 | 输出 | show_choice_ui() | 显示选择界面 |
| 存档系统 | 输出 | save_choice_history() | 保存选择历史 |

## Formulas

**选择影响力公式**：

`choice_impact = base_value × consequence_multiplier × time_decay`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| base_value | - | int | 1-10 | 选择基础影响力 |
| consequence_multiplier | - | float | 0.5-2.0 | 后果乘数 |
| time_decay | - | float | 0.8-1.0 | 时间衰减 |

**输出范围**：0.4到20
**示例**：选择拯救NPC base_value=8，consequence_multiplier=1.5，time_decay=0.9 → impact=10.8

**选择权重公式**：

`choice_weight = (moral_value + strategic_value + narrative_value) / 3`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| moral_value | - | int | -10~+10 | 道德价值（善良/邪恶） |
| strategic_value | - | int | -10~+10 | 策略价值（有利/不利） |
| narrative_value | - | int | -10~+10 | 叙事价值（重要/不重要） |

**输出范围**：-10到+10
**示例**：选择帮助村民 moral=+8，strategic=-3，narrative=+5 → weight=+3.33

**选择解锁公式**：

`unlock_condition = (required_marks_met) AND (required_choices_met) AND (required_items_met)`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| required_marks_met | - | bool | true/false | 是否满足印记要求 |
| required_choices_met | - | bool | true/false | 是否满足选择要求 |
| required_items_met | - | bool | true/false | 是否满足物品要求 |

**输出范围**：true/false
**示例**：解锁隐藏区域需要善良印记≥3 AND 已选择帮助村民 AND 拥有钥匙

## Edge Cases

| # | 边界情况 | 处理方式 |
|---|----------|----------|
| 1 | **选择超时（对话中未选择）** | 自动选择默认选项（通常是"沉默"） |
| 2 | **选择冲突（两个选择互斥）** | 后选择覆盖前选择，提示冲突 |
| 3 | **选择导致死锁（无法继续）** | 提供"后悔药"机制（消耗资源） |
| 4 | **选择影响已解锁内容** | 已解锁内容保持解锁，但可能改变状态 |
| 5 | **选择导致NPC死亡** | NPC死亡后相关剧情终止，但印记保留 |
| 6 | **选择导致卡牌池变化** | 已获得的卡牌保留，但新卡牌池改变 |
| 7 | **选择导致规则冲突** | 规则改写系统处理冲突（优先级机制） |
| 8 | **选择导致世界状态异常** | 世界状态系统验证状态合法性 |

## Tuning Knobs

| 调优参数 | 默认值 | 安全范围 | 极端行为 |
|----------|--------|----------|----------|
| 选择时间限制 | 30秒 | 10-60秒 | 太短=压力大，太长=无紧迫感 |
| 选择影响力基础值 | 5 | 1-10 | 太低=无感，太高=过于敏感 |
| 印记叠加上限 | 10 | 5-20 | 太少=策略受限，太多=管理复杂 |
| 元素切换冷却 | 5分钟 | 1-10分钟 | 太短=无策略，太长=过于限制 |
| 规则改写上限 | 3 | 1-5 | 太少=选择受限，太多=规则混乱 |
| 选择后悔消耗 | 100金币 | 50-500金币 | 太低=无代价，太高=不敢尝试 |
| 隐藏内容解锁阈值 | 3印记 | 1-5印记 | 太低=太容易，太高=不可达 |
| 选择后果延迟 | 0-3关 | 0-5关 | 太短=无悬念，太长=忘记选择 |

## Visual/Audio Requirements

**视觉效果**：
- **选择界面**：卡牌风格的选择卡片，带发光边框
- **选择确认**：选中卡片放大 + 光效爆发
- **印记获得**：印记图标从选择点飞向玩家头像
- **元素解锁**：元素图标旋转出现 + 特效
- **规则改变**：规则文字从屏幕上方飘落
- **后果显现**：屏幕渐变 + 回忆闪回效果

**音频效果**：
- **选择出现**：悬疑音效
- **选择确认**：确认音效 + 印记获得音效
- **元素解锁**：元素音效
- **规则改变**：规则改写音效
- **后果显现**：回忆音效 + 情感音乐

## UI Requirements

**选择界面布局**：
```
┌─────────────────────────────────────────┐
│  选择背景（场景/对话）                   │
├─────────────────────────────────────────┤
│                                         │
│  选择卡片区域（2-4个选项）               │
│  ┌─────┐  ┌─────┐  ┌─────┐             │
│  │  A  │  │  B  │  │  C  │             │
│  └─────┘  └─────┘  └─────┘             │
│                                         │
├─────────────────────────────────────────┤
│  选择提示（选择后果预览）                │
├─────────────────────────────────────────┤
│  当前印记状态（善良/邪恶/中立）          │
└─────────────────────────────────────────┘
```

**交互规则**：
- 鼠标点击选择卡片 → 选中选项
- 键盘1/2/3 → 快速选择
- 选择确认后不可撤销
- 选择超时自动选择默认选项

## Acceptance Criteria

| # | 验收标准 | Given-When-Then |
|---|----------|-----------------|
| AC-1 | 选择触发 | GIVEN 玩家到达选择点 WHEN 对话/剧情进行到选择点 THEN 选择界面正确显示 |
| AC-2 | 选择确认 | GIVEN 玩家做出选择 WHEN 确认选择 THEN 选择效果正确应用 |
| AC-3 | 印记获得 | GIVEN 选择留下印记 WHEN 选择确认 THEN 印记正确添加到玩家 |
| AC-4 | 元素解锁 | GIVEN 选择解锁元素 WHEN 选择确认 THEN 元素正确解锁 |
| AC-5 | 规则改变 | GIVEN 选择改变规则 WHEN 选择确认 THEN 规则正确改写 |
| AC-6 | 选择历史 | GIVEN 玩家做出选择 WHEN 选择完成 THEN 选择正确保存到历史 |
| AC-7 | 选择超时 | GIVEN 选择有时间限制 WHEN 超时未选择 THEN 自动选择默认选项 |
| AC-8 | 选择冲突 | GIVEN 两个选择互斥 WHEN 后选择覆盖前选择 THEN 提示冲突并应用新选择 |
| AC-9 | 后果显现 | GIVEN 选择有延迟后果 WHEN 到达后续剧情 THEN 后果正确显现 |
| AC-10 | 选择界面 | GIVEN 到达选择点 WHEN 显示选择界面 THEN 界面清晰显示所有选项 |
