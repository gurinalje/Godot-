# Area System

> **Status**: In Design
> **Author**: User + Agents
> **Last Updated**: 2026-06-03
> **Implements Pillar**: Pillar 3 (Explore = Reward), Pillar 4 (Persistent Growth)

## Overview

区域系统是《命运卡牌局》的空间管理层，负责定义每个世界中的区域结构、内容和交互。系统管理区域内的NPC、敌人、宝箱、事件等元素，为玩家提供丰富的探索体验。

## Player Fantasy

**"每个区域都有独特的惊喜等待发现"**

玩家应该感受到：
- **探索欲**：每个角落都可能有隐藏内容
- **沉浸感**：区域的视觉和音效营造独特氛围
- **策略性**：区域布局影响我的探索路线
- **奖励感**：探索区域获得卡牌、金币、道具
- **安全感**：区域有明确的边界和出口

**参考体验**：空洞骑士的区域设计 + 塞尔达传说的迷宫探索 + 杀戮尖塔的节点设计

## Detailed Design

### Core Rules

**区域结构**：
- 每个区域有独立的地图（2D平面）
- 区域内有多个节点（战斗/事件/宝箱/NPC/出口）
- 节点之间有连接路径
- 玩家从入口节点开始，探索到出口节点

**节点类型**：
- **战斗节点**：与敌人战斗
- **事件节点**：触发剧情事件
- **宝箱节点**：获得奖励
- **NPC节点**：与NPC对话
- **商店节点**：购买卡牌/道具
- **休息节点**：恢复生命/保存
- **隐藏节点**：需要特殊条件发现
- **Boss节点**：区域Boss战斗

**区域规则**：
- 玩家只能移动到相邻节点
- 已访问的节点标记为已探索
- 隐藏节点需要特定条件才能发现
- Boss节点需要完成前置节点才能挑战

### States and Transitions

| 当前状态 | 触发条件 | 目标状态 | 说明 |
|----------|----------|----------|------|
| 区域未进入 | 玩家进入区域 | 探索中 | 加载区域地图 |
| 探索中 | 玩家移动到节点 | 节点交互 | 触发节点事件 |
| 节点交互 | 交互完成 | 探索中 | 返回地图 |
| 探索中 | 到达出口 | 区域完成 | 标记区域完成 |
| 区域完成 | 重新进入 | 探索中 | 允许重复探索 |
| 隐藏节点 | 发现条件满足 | 节点交互 | 发现隐藏内容 |

### Interactions with Other Systems

| 交互系统 | 方向 | 接口 | 说明 |
|----------|------|------|------|

## Formulas

**节点连接公式**：

`node_connected = (distance <= max_distance) AND (path_exists)`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| distance | - | float | 0-100 | 节点间距离 |
| max_distance | - | float | 50 | 最大连接距离 |
| path_exists | - | bool | true/false | 路径是否存在 |

**输出范围**：true/false
**示例**：两个节点距离30，max_distance=50 → 可连接

**隐藏节点发现公式**：

`hidden_found = (discovery_stat >= threshold) AND (proximity_check)`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| discovery_stat | - | int | 0-100 | 玩家发现属性 |
| threshold | - | int | 50-90 | 发现阈值 |
| proximity_check | - | bool | true/false | 是否在发现范围内 |

**输出范围**：true/false
**示例**：玩家发现属性70，threshold=60，在范围内 → 发现隐藏节点

**区域完成度公式**：

`area_completion = (explored_nodes / total_nodes) × 100%`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| explored_nodes | - | int | 0-20 | 已探索节点数 |
| total_nodes | - | int | 5-20 | 总节点数 |

**输出范围**：0%到100%
**示例**：区域有10个节点，已探索7个 → completion=70%

## Edge Cases

| # | 边界情况 | 处理方式 |
|---|----------|----------|
| 1 | **所有节点已探索** | 显示"区域完成"，解锁出口 |
| 2 | **节点连接断裂** | 验证连接完整性，修复断裂 |
| 3 | **隐藏节点已发现** | 标记为已发现，不再隐藏 |
| 4 | **Boss节点未解锁** | 显示解锁条件，阻止进入 |
| 5 | **区域重复进入** | 允许重复进入，保留探索进度 |
| 6 | **节点事件失败** | 允许重试，不消耗资源 |
| 7 | **区域状态异常** | 验证状态合法性，修复异常 |
| 8 | **节点奖励已领取** | 标记为已领取，不再发放 |

## Tuning Knobs

| 调优参数 | 默认值 | 安全范围 | 极端行为 |
|----------|--------|----------|----------|
| 节点数量/区域 | 10 | 5-20 | 太少=内容不足，太多=过于复杂 |
| 隐藏节点比例 | 15% | 5-25% | 太少=无惊喜，太多=过于分散 |
| 战斗节点比例 | 40% | 20-60% | 太少=战斗不足，太多=过于重复 |
| 事件节点比例 | 20% | 10-30% | 太少=剧情不足，太多=打断节奏 |
| 宝箱节点比例 | 15% | 5-25% | 太少=奖励不足，太多=过于容易 |
| 节点连接密度 | 2.5 | 1.5-3.5 | 太低=路径单一，太高=过于复杂 |
| 区域大小 | 中型 | 小型/中型/大型 | 影响探索时间和内容密度 |
| Boss难度 | 普通 | 简单/普通/困难 | 影响区域挑战性 |

## Visual/Audio Requirements

**视觉效果**：
- **区域地图**：手绘风格的节点地图，带连接线
- **节点图标**：不同类型的节点有不同图标
- **探索进度**：已探索节点高亮显示
- **隐藏节点**：特殊光效表示隐藏节点
- **Boss节点**：特殊图标和光效
- **区域背景**：每个区域有独特的背景风格

**音频效果**：
- **节点交互**：不同节点有不同音效
- **隐藏发现**：发现音效 + 惊喜音乐
- **Boss战斗**：Boss战斗音乐
- **区域进入**：区域环境音效
- **探索脚步**：移动音效

## UI Requirements

**区域地图界面**：
```
┌─────────────────────────────────────────┐
│  区域名称 + 完成度                      │
├─────────────────────────────────────────┤
│                                         │
│  节点地图（节点图标 + 连接线）          │
│  ┌─────┐    ┌─────┐    ┌─────┐         │
│  │ 入口 │────│ 战斗 │────│ 事件 │         │
│  └─────┘    └─────┘    └─────┘         │
│      │          │          │             │
│      └──────┬───┴──────────┘             │
│           ┌─────┐                       │
│           │ 宝箱 │                       │
│           └─────┘                       │
│              │                          │
│           ┌─────┐                       │
│           │ Boss │                       │
│           └─────┘                       │
├─────────────────────────────────────────┤
│  当前位置 + 可移动节点                  │
├─────────────────────────────────────────┤
│  区域信息（描述/危险等级/推荐等级）     │
└─────────────────────────────────────────┘
```

**交互规则**：
- 点击节点 → 移动到节点
- 点击已探索节点 → 查看节点详情
- 点击隐藏节点（如已发现）→ 移动到隐藏节点
- 拖拽地图 → 移动视图
- 缩放 → 放大/缩小地图

## Acceptance Criteria

| # | 验收标准 | Given-When-Then |
|---|----------|-----------------|
| AC-1 | 区域加载 | GIVEN 玩家进入区域 WHEN 区域加载 THEN 地图正确显示所有节点 |
| AC-2 | 节点移动 | GIVEN 玩家点击节点 WHEN 移动判定 THEN 玩家正确移动到节点 |
| AC-3 | 节点交互 | GIVEN 玩家到达节点 WHEN 触发交互 THEN 节点事件正确执行 |
| AC-4 | 隐藏发现 | GIVEN 满足发现条件 WHEN 接近隐藏节点 THEN 隐藏节点正确显示 |
| AC-5 | 区域完成 | GIVEN 完成所有节点 WHEN 到达出口 THEN 区域完成动画显示 |
| AC-6 | 状态保存 | GIVEN 区域状态变更 WHEN 保存游戏 THEN 状态正确保存 |
| AC-7 | 状态加载 | GIVEN 加载存档 WHEN 游戏启动 THEN 区域状态正确恢复 |
| AC-8 | 地图显示 | GIVEN 打开区域地图 WHEN 地图加载 THEN 所有节点正确显示 |
| AC-9 | 节点奖励 | GIVEN 到达宝箱节点 WHEN 领取奖励 THEN 奖励正确发放 |
| AC-10 | Boss挑战 | GIVEN 完成前置节点 WHEN 挑战Boss THEN Boss战斗正确开始 |
