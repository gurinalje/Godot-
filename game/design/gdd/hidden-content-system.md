# Hidden Content System

> **Status**: Designed
> **Author**: User + Agents
> **Last Updated**: 2026-06-03
> **Implements Pillar**: Pillar 3 (Explore = Reward)

## Overview

隐藏内容系统是《命运卡牌局》的彩蛋管理层，负责管理游戏中的隐藏Boss、稀有卡牌、秘密区域和彩蛋内容。系统为探索型玩家提供额外的挑战和奖励，增加游戏的重玩价值。

## Player Fantasy

**"我在发现游戏中的秘密，每个发现都让我惊喜"**

玩家应该感受到：
- **惊喜感**：发现隐藏内容的意外喜悦
- **成就感**：挑战隐藏Boss的胜利满足
- **收集欲**：收集稀有卡牌的渴望
- **探索欲**：探索每个角落的动力
- **独特感**：拥有别人没有的稀有物品

**参考体验**：空洞骑士的隐藏Boss + 传说之下的彩蛋 + 杀戮尖塔的隐藏事件

## Detailed Design

### Core Rules

**隐藏内容类型**：
- **隐藏Boss**：需要特殊条件触发的强大敌人
- **稀有卡牌**：极低概率掉落的传说卡牌
- **秘密区域**：需要特定条件发现的隐藏地图
- **彩蛋事件**：致敬其他游戏的特殊事件
- **隐藏NPC**：需要特殊条件出现的NPC

**触发条件**：
- **印记触发**：获得特定印记组合
- **选择触发**：做出特定选择序列
- **探索触发**：在特定时间访问特定地点
- **物品触发**：拥有特定物品组合
- **成就触发**：完成特定成就

**发现机制**：
- **概率发现**：随机概率触发
- **条件发现**：满足所有条件后触发
- **线索发现**：通过线索推理发现
- **社区发现**：通过玩家社区分享发现

### States and Transitions

| 当前状态 | 触发条件 | 目标状态 | 说明 |
|----------|----------|----------|------|
| 隐藏 | 满足触发条件 | 已发现 | 发现隐藏内容 |
| 已发现 | 挑战/收集 | 已完成 | 完成隐藏内容 |
| 已完成 | 重新触发 | 已发现 | 重复体验 |
| 未触发 | 线索收集 | 线索中 | 收集线索 |
| 线索中 | 线索完整 | 已发现 | 通过线索发现 |

## Formulas

**隐藏发现公式**：

`hidden_discovered = (all_conditions_met) AND (discovery_chance_roll)`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| all_conditions_met | - | bool | true/false | 是否满足所有条件 |
| discovery_chance_roll | - | bool | true/false | 发现概率判定 |

**输出范围**：true/false
**示例**：满足所有印记和选择要求 AND 概率判定通过 → 发现隐藏Boss

**稀有卡牌掉落公式**：

`rare_card_dropped = (base_chance × luck_modifier × difficulty_modifier) >= random_roll`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| base_chance | - | float | 0.01-0.1 | 基础掉落概率 |
| luck_modifier | - | float | 0.5-2.0 | 幸运修正 |
| difficulty_modifier | - | float | 0.8-1.5 | 难度修正 |
| random_roll | - | float | 0-1 | 随机数 |

**输出范围**：true/false
**示例**：base_chance=0.05，luck_modifier=1.5，difficulty_modifier=1.2，random_roll=0.08 → dropped (0.09 > 0.08)

**隐藏内容完成度公式**：

`hidden_completion = (discovered_hidden / total_hidden) × 100%`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| discovered_hidden | - | int | 0-20 | 已发现隐藏内容数 |
| total_hidden | - | int | 5-20 | 总隐藏内容数 |

**输出范围**：0%到100%
**示例**：游戏有15个隐藏内容，已发现9个 → completion=60%

## Edge Cases

| # | 边界情况 | 处理方式 |
|---|----------|----------|
| 1 | **隐藏内容已发现** | 标记为已发现，不再触发 |
| 2 | **条件冲突** | 优先满足最近的条件 |
| 3 | **概率永远不触发** | 设置保底机制（100次后必触发） |
| 4 | **隐藏Boss太难** | 提供难度调整机制 |
| 5 | **稀有卡牌已拥有** | 转化为金币或其他资源 |
| 6 | **隐藏内容bug** | 提供跳过机制 |
| 7 | **隐藏状态异常** | 验证状态合法性，修复异常 |
| 8 | **隐藏内容重复触发** | 允许重复体验，保留完成状态 |

## Tuning Knobs

| 调优参数 | 默认值 | 安全范围 | 极端行为 |
|----------|--------|----------|----------|
| 隐藏Boss数量 | 5 | 3-10 | 太少=挑战不足，太多=过于分散 |
| 稀有卡牌数量 | 10 | 5-20 | 太少=收集不足，太多=过于容易 |
| 秘密区域数量 | 3 | 2-5 | 太少=探索不足，太多=过于分散 |
| 彩蛋事件数量 | 5 | 3-10 | 太少=惊喜不足，太多=过于分散 |
| 基础发现概率 | 5% | 1-10% | 太低=不可发现，太高=无惊喜 |
| 保底次数 | 100 | 50-200 | 太低=无挑战，太高=过于困难 |
| 隐藏Boss难度 | 困难 | 普通/困难/地狱 | 影响隐藏Boss挑战性 |
| 稀有卡牌掉率 | 5% | 1-10% | 太低=不可获得，太高=过于容易 |

## Acceptance Criteria

| # | 验收标准 | Given-When-Then |
|---|----------|-----------------|
| AC-1 | 隐藏发现 | GIVEN 满足发现条件 WHEN 发现判定通过 THEN 隐藏内容正确显示 |
| AC-2 | 隐藏Boss | GIVEN 触发隐藏Boss WHEN 战斗开始 THEN Boss战斗正确开始 |
| AC-3 | 稀有卡牌 | GIVEN 击败隐藏Boss WHEN 掉落判定 THEN 稀有卡牌正确掉落 |
| AC-4 | 秘密区域 | GIVEN 发现秘密区域 WHEN 进入判定 THEN 正确进入秘密区域 |
| AC-5 | 彩蛋事件 | GIVEN 触发彩蛋事件 WHEN 事件开始 THEN 彩蛋事件正确执行 |
| AC-6 | 隐藏NPC | GIVEN 触发隐藏NPC WHEN NPC出现 THEN NPC正确显示 |
| AC-7 | 线索收集 | GIVEN 收集线索 WHEN 线索更新 THEN 线索正确记录 |
| AC-8 | 状态保存 | GIVEN 隐藏状态变更 WHEN 保存游戏 THEN 状态正确保存 |
| AC-9 | 状态加载 | GIVEN 加载存档 WHEN 游戏启动 THEN 隐藏状态正确恢复 |
| AC-10 | 完成度显示 | GIVEN 查看隐藏完成度 WHEN 显示完成度 THEN 百分比正确计算 |
