# 卡牌战斗原型

> **PROTOTYPE - NOT FOR PRODUCTION**  
> **Question**: 卡牌战斗系统的"爽快感"是否可行？  
> **Date**: 2026-06-03

## 概述

这是一个最小可行原型，用于验证卡牌战斗系统的核心机制：
- 卡牌打出的手感（即时反馈）
- Combo连锁的触发机制
- 伤害数字的视觉反馈
- 能量系统的策略深度

## 文件结构

```
prototypes/card-battle/
├─ battle_scene.gd    # 战斗逻辑脚本
├─ battle_scene.tscn  # 战斗场景
└─ README.md          # 本文件
```

## 运行方式

1. 在Godot中打开项目
2. 运行 `prototypes/card-battle/battle_scene.tscn`

## 测试内容

### 卡牌类型
- **打击** (1能量) - 造成6点伤害
- **防御** (1能量) - 获得5点护甲
- **火球术** (2能量) - 造成10点火焰伤害
- **治疗** (1能量) - 恢复5点生命
- **连击** (1能量) - 造成3×连击数伤害

### 核心机制
1. **能量系统** - 每回合3点能量，合理分配
2. **手牌管理** - 每回合抽2张牌
3. **Combo连锁** - 连续使用连击卡，伤害递增
4. **即时反馈** - 伤害数字弹出、屏幕震动
