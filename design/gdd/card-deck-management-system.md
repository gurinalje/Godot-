# Card Deck Management System

> **Status**: Designed
> **Author**: User + Agents
> **Last Updated**: 2026-06-03
> **Implements Pillar**: Pillar 2 (Satisfying Depth), Pillar 4 (Persistent Growth)

## Overview

卡组管理系统是《命运卡牌局》的卡牌收集层，负责管理玩家的卡牌收集、升级和构筑。系统连接卡组构筑系统和卡牌升级系统，为玩家提供完整的卡牌管理体验。

## Player Fantasy

**"我在收集和管理一副强大的卡牌收藏"**

玩家应该感受到：
- **收集欲**：收集所有卡牌的满足感
- **管理感**：管理卡牌收藏的乐趣
- **成就感**：构建出强力卡组的满足感
- **成长感**：卡牌收藏在不断壮大
- **个性化**：我的卡牌收藏反映了我的游戏风格

**参考体验**：游戏王的卡牌收集 + 炉石传说的卡牌管理 + 杀戮尖塔的卡组构筑

## Detailed Design

### Core Rules

**卡牌收集**：
- 玩家通过战斗、任务、商店获得卡牌
- 卡牌有稀有度（普通/稀有/史诗/传说）
- 卡牌有元素属性（火/水/风/土/雷）
- 卡牌有类型（召唤/伤害/环境/增益）

**卡牌管理**：
- 查看所有已收集的卡牌
- 按稀有度/元素/类型筛选卡牌
- 按名称/稀有度/获得时间排序卡牌
- 搜索特定卡牌

**卡牌升级**：
- 消耗金币和相同卡牌升级卡牌
- 升级提升卡牌属性
- 最高可升级到5级

**卡组构筑**：
- 从收藏中选择卡牌构建卡组
- 卡组有大小限制（20-40张）
- 每张卡牌最多携带3张
- 可保存多个卡组配置

## Formulas

**卡牌收集完成度公式**：

`collection_completion = (collected_cards / total_cards) × 100%`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| collected_cards | - | int | 0-200 | 已收集卡牌数 |
| total_cards | - | int | 50-200 | 总卡牌数 |

**输出范围**：0%到100%
**示例**：游戏有100张卡牌，已收集65张 → completion=65%

**卡牌稀有度分布公式**：

`rarity_distribution = (common_ratio × 0.6) + (rare_ratio × 0.3) + (epic_ratio × 0.08) + (legendary_ratio × 0.02)`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| common_ratio | - | float | 0-1 | 普通卡牌比例 |
| rare_ratio | - | float | 0-1 | 稀有卡牌比例 |
| epic_ratio | - | float | 0-1 | 史诗卡牌比例 |
| legendary_ratio | - | float | 0-1 | 传说卡牌比例 |

**输出范围**：0到1
**示例**：common=0.6，rare=0.3，epic=0.08，legendary=0.02 → distribution=1.0

**卡组协同公式**：

`deck_synergy = (element_synergy × 0.4) + (type_synergy × 0.3) + (effect_synergy × 0.3)`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| element_synergy | - | float | 0-1 | 元素协同比例 |
| type_synergy | - | float | 0-1 | 类型协同比例 |
| effect_synergy | - | float | 0-1 | 效果协同比例 |

**输出范围**：0到1
**示例**：element=0.8，type=0.7，effect=0.6 → synergy=0.71

## Edge Cases

| # | 边界情况 | 处理方式 |
|---|----------|----------|
| 1 | **收藏为空** | 提示"收藏为空"，阻止查看 |
| 2 | **卡牌已满级** | 提示"已满级"，阻止升级 |
| 3 | **卡牌数量超限** | 提示"卡牌数量超限"，阻止获取 |
| 4 | **卡牌被删除** | 从收藏移除已删除卡牌 |
| 5 | **卡牌升级失败** | 提示"升级失败"，退还材料 |
| 6 | **卡组配置丢失** | 从存档恢复配置 |
| 7 | **收藏状态异常** | 验证状态合法性，修复异常 |
| 8 | **卡牌重复获得** | 增加卡牌数量，不重复显示 |

## Tuning Knobs

| 调优参数 | 默认值 | 安全范围 | 极端行为 |
|----------|--------|----------|----------|
| 总卡牌数量 | 100 | 50-200 | 太少=收集不足，太多=过于复杂 |
| 普通卡牌比例 | 60% | 50-70% | 太少=稀有度失衡，太多=无稀有感 |
| 稀有卡牌比例 | 30% | 20-40% | 太少=稀有度失衡，太多=过于容易 |
| 史诗卡牌比例 | 8% | 5-12% | 太少=稀有度失衡，太多=过于容易 |
| 传说卡牌比例 | 2% | 1-3% | 太少=稀有度失衡，太多=过于容易 |
| 卡牌获取概率 | 10% | 5-20% | 太低=收集困难，太高=过于容易 |
| 卡牌升级消耗 | 100金币 | 50-200 | 太低=太容易，太高=过于困难 |
| 卡牌收藏上限 | 200 | 100-500 | 太少=收集受限，太多=管理复杂 |

## Visual/Audio Requirements

**视觉效果**：
- **收藏界面**：卡牌网格 + 筛选/排序
- **卡牌详情**：卡牌立绘 + 属性展示
- **卡牌获取**：卡牌获得动画
- **卡牌升级**：升级动画 + 属性提升
- **稀有度标识**：不同稀有度有不同边框
- **元素标识**：不同元素有不同颜色

**音频效果**：
- **卡牌获取**：获得音效（稀有度不同音效不同）
- **卡牌升级**：升级音效
- **卡牌筛选**：筛选音效
- **卡牌排序**：排序音效
- **卡牌详情**：详情音效
