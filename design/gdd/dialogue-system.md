## Overview

**对话系统**是游戏的叙事核心，负责管理NPC对话、玩家选择和剧情分支。它定义了对话树的结构、选择的影响、分支的触发条件，为选择系统、故事印记系统和规则改写系统提供对话数据。玩家通过对话系统感受到"我的选择真的改变了故事"——每个对话选择都可能影响剧情走向、NPC态度和游戏规则。

对话系统读取对话数据文件，结合选择系统和故事印记系统，实现对话的分支和影响。玩家通过对话系统感受到"我的选择真的改变了故事的走向，每个对话都有意义"。

## Player Fantasy

**对话系统**是玩家直接体验的系统——玩家通过对话与NPC互动，做出选择，影响故事走向。

**情感目标**："我的选择真的改变了故事"

**玩家应该感受到**：
- **掌控感**：我的选择真的改变了故事的走向
- **沉浸感**：我与NPC的对话是有意义的，不是无意义的点击
- **惊喜感**：我的选择解锁了新的剧情分支和隐藏内容

**情感锚点**：像传说之下中每个对话选择都影响故事的沉浸感——"我的选择真的改变了这个世界！"

## Detailed Design

### Core Rules

1. **对话树结构**：
   - **节点**：对话的基本单位，包含NPC台词和玩家选项
   - **分支**：玩家选择导致的不同对话路径
   - **条件**：触发分支的条件（如故事印记、规则改写、NPC态度）
   - **结果**：对话选择的后果（如获得印记、改变NPC态度、解锁剧情）

2. **对话类型**：
   - **普通对话**：NPC与玩家的日常对话
   - **任务对话**：与任务相关的对话
   - **剧情对话**：与主线/支线剧情相关的对话
   - **隐藏对话**：需要特定条件才能触发的对话

3. **选择影响**：
   - **即时影响**：选择立即改变当前对话
   - **延迟影响**：选择在后续对话中产生后果
   - **规则影响**：选择改变游戏规则（通过规则改写系统）
   - **印记影响**：选择留下故事印记（通过故事印记系统）

4. **NPC态度系统**：
   - **友好**：NPC对玩家友好，提供帮助
   - **中立**：NPC对玩家中立，提供信息
   - **敌对**：NPC对玩家敌对，提供挑战
   - **隐藏**：NPC态度需要特定条件才能解锁

5. **对话触发条件**：
   - **位置触发**：玩家到达特定位置时触发
   - **时间触发**：游戏时间到达特定时间时触发
   - **印记触发**：玩家拥有特定故事印记时触发
   - **规则触发**：游戏规则被改写时触发

### States and Transitions

| 状态 | 描述 | 触发条件 |
|------|------|----------|
| **空闲** | NPC等待玩家互动 | 玩家接近NPC时进入 |
| **对话中** | NPC与玩家正在对话 | 玩家点击NPC时进入 |
| **选择中** | 玩家正在做出选择 | 对话到达选择节点时进入 |
| **结果中** | 对话选择的结果正在显示 | 玩家做出选择后进入 |
| **结束** | 对话结束 | 对话树遍历完成后进入 |

**状态转换规则**：
- 空闲 → 对话中：玩家点击NPC时自动进入
- 对话中 → 选择中：对话到达选择节点时自动进入
- 选择中 → 结果中：玩家做出选择后自动进入
- 结果中 → 对话中：结果显示完成后自动进入（如果还有后续对话）
- 结果中 → 结束：结果显示完成后自动进入（如果对话结束）

### Interactions with Other Systems

| 系统 | 数据流向 | 接口描述 |
|------|----------|----------|
| **选择系统** | 输出 | 提供对话选择数据，触发选择事件 |
| **故事印记系统** | 输出 | 提供故事印记数据，触发印记事件 |
| **规则改写系统** | 输出 | 提供规则改写数据，触发规则改写事件 |

| **任务系统** | 输入 | 读取任务数据，触发任务相关对话 |

## Formulas

### 对话好感度公式

`affinity = base_affinity + (good_choices × affinity_gain) - (bad_choices × affinity_loss)`

**变量：**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_affinity | A_base | int | 0-50 | NPC基础好感度 |
| good_choices | G_c | int | 0-100 | 好的选择数量 |
| affinity_gain | A_gain | int | 1-10 | 每个好选择增加的好感度 |
| bad_choices | B_c | int | 0-100 | 坏的选择数量 |
| affinity_loss | A_loss | int | 1-10 | 每个坏选择减少的好感度 |
| affinity | A | int | -100-100 | 最终好感度 |

**输出范围：** -100到100（正常游戏范围）
**示例：** 基础好感度20，3个好选择（+5），1个坏选择（-3），好感度 = 20 + (3×5) - (1×3) = 32

### 对话解锁条件公式

`unlock = (affinity >= threshold) AND (marks >= mark_threshold) AND (choices >= choice_threshold)`

**变量：**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| affinity | A | int | -100-100 | NPC好感度 |
| threshold | T | int | 0-100 | 好感度阈值 |
| marks | M | int | 0-10 | 故事印记数量 |
| mark_threshold | M_t | int | 0-10 | 故事印记阈值 |
| choices | C | int | 0-100 | 选择数量 |
| choice_threshold | C_t | int | 0-100 | 选择阈值 |
| unlock | U | bool | true/false | 是否解锁 |

**输出范围：** true或false
**示例：** 好感度32 >= 30，故事印记5 >= 3，选择10 >= 5，解锁 = true

### 对话分支权重公式

`branch_weight = base_weight + (affinity × affinity_factor) + (marks × mark_factor)`

**变量：**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_weight | W_base | int | 1-10 | 基础权重 |
| affinity | A | int | -100-100 | NPC好感度 |
| affinity_factor | A_f | float | 0-0.1 | 好感度系数 |
| marks | M | int | 0-10 | 故事印记数量 |
| mark_factor | M_f | float | 0-0.1 | 故事印记系数 |
| branch_weight | W | int | 1-20 | 最终权重 |

**输出范围：** 1到20（正常游戏范围）
**示例：** 基础权重5，好感度32×0.05，故事印记5×0.1，权重 = 5 + (32×0.05) + (5×0.1) = 7.6

## Edge Cases

- **如果NPC好感度达到上限（100）**：好感度不再增加，显示"好感度已满"提示。
- **如果NPC好感度达到下限（-100）**：好感度不再减少，显示"好感度已最低"提示。
- **如果对话选择导致NPC态度变化**：NPC态度立即更新，显示新的态度图标。
- **如果对话选择导致故事印记获得**：故事印记立即获得，显示获得提示。
- **如果对话选择导致规则改写**：规则改写立即生效，显示规则改写提示。
- **如果对话选择导致剧情分支**：剧情立即分支，显示分支提示。
- **如果对话选择导致隐藏剧情触发**：隐藏剧情立即触发，显示隐藏剧情提示。
- **如果对话选择导致NPC离开**：NPC立即离开，显示NPC离开提示。

## Visual/Audio Requirements

### 视觉效果
- **对话框动画**：对话框从屏幕底部滑入，NPC头像从左侧滑入
- **选择按钮动画**：选择按钮从屏幕底部滑入，鼠标悬停时放大
- **好感度变化动画**：好感度变化时，数字跳动，颜色变化（正数绿色，负数红色）
- **故事印记获得动画**：故事印记获得时，印记图标从屏幕中央浮现，带有发光效果
- **规则改写激活动画**：规则改写激活时，规则改写图标从屏幕中央浮现，带有旋转效果
- **NPC态度变化动画**：NPC态度变化时，态度图标跳动，颜色变化

### 音频效果
- **对话开始音效**：对话开始时，播放对话开始音效（叮咚声）
- **对话结束音效**：对话结束时，播放对话结束音效（叮咚声）
- **选择音效**：选择选项时，播放选择音效（点击声）
- **好感度变化音效**：好感度变化时，播放好感度变化音效（上升/下降音效）
- **故事印记获得音效**：故事印记获得时，播放故事印记获得音效（叮咚声）
- **规则改写激活音效**：规则改写激活时，播放规则改写激活音效（旋转声）
- **NPC态度变化音效**：NPC态度变化时，播放NPC态度变化音效（叮咚声）

## UI Requirements

### 对话框
- **位置**：屏幕底部，占屏幕宽度的80%
- **内容**：NPC头像、NPC名称、对话文本
- **大小**：宽度80%屏幕，高度20%屏幕
- **动画**：对话框从屏幕底部滑入，NPC头像从左侧滑入

### 选择按钮
- **位置**：屏幕底部，对话框下方
- **内容**：选择选项文本
- **大小**：每个选项宽度80%屏幕，高度10%屏幕
- **动画**：选择按钮从屏幕底部滑入，鼠标悬停时放大

### 好感度显示
- **位置**：对话框右上角
- **内容**：NPC好感度数值
- **大小**：50x50像素
- **动画**：好感度变化时，数字跳动，颜色变化

### 故事印记显示
- **位置**：对话框左上角
- **内容**：故事印记图标
- **大小**：30x30像素
- **动画**：故事印记获得时，印记图标从屏幕中央浮现

### 规则改写显示
- **位置**：对话框左上角，故事印记图标下方
- **内容**：规则改写图标
- **大小**：30x30像素
- **动画**：规则改写激活时，规则改写图标从屏幕中央浮现
