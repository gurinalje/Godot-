# 存档系统

> **Status**: In Design  
> **Author**: User + AI  
> **Last Updated**: 2026-06-03  
> **Implements Pillar**: 持久的成长感

---

## Overview

存档系统是游戏的持久化基础设施，负责将玩家的游戏进度、选择结果、卡牌收集、世界状态等数据序列化并存储到本地文件。系统支持自动保存（关键节点）和手动保存（玩家触发），确保玩家在任何中断后都能恢复到之前的状态。存档数据采用JSON格式，便于调试和未来扩展。

## Player Fantasy

存档系统应该让玩家感受到"我的进度是安全的"。当玩家看到"游戏已保存"的提示时，他们应该感到安心，知道自己的选择、收集和成就都被妥善保存。这个系统不是玩家会"爱用"的系统，而是玩家会"信任"的系统——就像银行保险箱，你不会欣赏它的设计，但你会信任它保护你的财产。

## Detailed Design

### Core Rules

1. **存档数据结构**
   - 每个存档包含：元数据（时间、游戏时长、位置）+ 游戏状态数据
   - 游戏状态数据采用嵌套字典结构，按系统组织
   - 存档文件格式：JSON（便于调试和扩展）

2. **保存时机**
   - 自动保存触发点：进入新区域、做出重要选择、击败Boss、定期保存（每5分钟）
   - 手动保存：玩家在菜单中选择"保存游戏"，选择槽位
   - 退出保存：玩家退出游戏时，自动保存到自动存档槽

3. **存档槽管理**
   - 手动存档槽：3-5个，显示存档时间、游戏时长、当前位置缩略图
   - 自动存档槽：1个，只读，显示"自动保存"标记
   - 存档槽满时：提示玩家覆盖现有存档

4. **存档验证**
   - 每个存档包含校验和（CRC32或简单哈希）
   - 加载时验证数据完整性
   - 损坏的存档：提示错误，尝试恢复部分数据

### States and Transitions

| 状态 | 描述 | 转换条件 |
|------|------|----------|
| **空闲** | 等待保存/加载触发 | — |
| **保存中** | 正在序列化数据并写入文件 | 收到保存请求 |
| **加载中** | 正在读取文件并反序列化数据 | 收到加载请求 |
| **验证中** | 检查存档数据完整性 | 加载完成后 |
| **成功** | 操作完成，显示确认信息 | 保存/加载成功 |
| **错误** | 操作失败，显示错误信息 | 校验失败/文件损坏/磁盘空间不足 |

```
[空闲] → 保存请求 → [保存中] → 写入完成 → [成功] → 返回 [空闲]
[空闲] → 加载请求 → [加载中] → 读取完成 → [验证中] → 验证通过 → [成功] → 返回 [空闲]
[验证中] → 验证失败 → [错误] → 返回 [空闲]
[保存中] → 写入失败 → [错误] → 返回 [空闲]
```

### Interactions with Other Systems

| 系统 | 数据流向 | 接口描述 |
|------|----------|----------|
| **卡牌数据库** | 双向 | 保存/加载卡牌收藏、卡组配置、卡牌升级状态 |
| **角色属性系统** | 双向 | 保存/加载角色等级、属性值、技能树状态 |
| **选择系统** | 双向 | 保存/加载选择历史、故事印记、规则改写状态 |
| **世界状态系统** | 双向 | 保存/加载已解锁区域、世界进度、隐藏内容状态 |
| **UI系统** | 输出 | 提供存档/加载界面，显示存档列表和操作反馈 |
| **音频系统** | 输出 | 播放保存/加载成功/失败音效 |

**数据格式约定**：
- 每个系统实现 `save_data() -> Dictionary` 方法
- 每个系统实现 `load_data(data: Dictionary)` 方法
- 存档系统调用所有注册系统的 `save_data()` 方法，合并为完整存档
- 加载时，存档系统调用所有注册系统的 `load_data()` 方法，传递对应数据

## Formulas

| 公式 | 描述 | 变量 |
|------|------|------|
| `存档文件大小` | 预估单个存档文件大小 | 存档数据结构复杂度 |
| `校验和` | CRC32或简单哈希算法 | 存档数据字符串 |
| `自动保存间隔` | 定期自动保存的时间间隔 | 时间（秒） |

**存档文件大小估算**：
- 元数据：约 1KB
- 玩家属性：约 2KB
- 卡牌收藏（100张）：约 5KB
- 选择历史：约 3KB
- 世界状态：约 4KB
- **总计：约 15KB**（纯文本JSON，压缩后约5KB）

**自动保存间隔**：
- 默认：300秒（5分钟）
- 最小：60秒（1分钟）
- 最大：600秒（10分钟）

## Edge Cases

| 边界情况 | 处理方式 | 原因 |
|----------|----------|------|
| **存档文件损坏** | 显示错误信息，尝试加载自动存档 | 确保玩家不会因单个损坏文件丢失所有进度 |
| **磁盘空间不足** | 保存前检查可用空间，不足时警告玩家 | 避免写入半完成的文件 |
| **保存过程中退出** | 使用写入-重命名模式，确保原子性 | 避免文件损坏 |
| **多个存档同时损坏** | 提示玩家，提供"重置游戏"选项 | 最后防线 |
| **存档版本不兼容** | 尝试数据迁移，失败时提示玩家 | 支持游戏更新 |
| **自动保存被覆盖** | 自动保存使用独立槽位，不会被手动存档覆盖 | 保护自动存档 |
| **存档文件过大** | 监控文件大小，超过1MB时警告 | 防止性能问题 |

| 依赖系统 | 依赖类型 | 数据接口 | 是否已设计 |
|----------|----------|----------|------------|
| **卡牌数据库** | 硬依赖 | `save_data() -> Dictionary` / `load_data(data: Dictionary)` | ❌ 未设计 |

| **选择系统** | 硬依赖 | `save_data() -> Dictionary` / `load_data(data: Dictionary)` | ❌ 未设计 |
| **世界状态系统** | 硬依赖 | `save_data() -> Dictionary` / `load_data(data: Dictionary)` | ❌ 未设计 |
| **UI系统** | 软依赖 | 存档界面、加载界面、存档列表显示 | ❌ 未设计 |

**依赖说明**：
- 硬依赖：系统必须存在才能正常工作
- 软依赖：系统存在时增强体验，但核心功能可独立运行
- 所有依赖系统均未设计，存档系统需要定义清晰的接口契约

## Tuning Knobs

| 参数 | 默认值 | 最小值 | 最大值 | 说明 |
|------|--------|--------|--------|------|
| `auto_save_interval` | 300秒 | 60秒 | 600秒 | 自动保存的时间间隔 |
| `max_save_slots` | 5 | 3 | 10 | 手动存档槽位数量 |
| `save_file_format` | "json" | - | - | 存档文件格式（json/binary） |
| `compression_enabled` | true | - | - | 是否启用文件压缩 |
| `checksum_enabled` | true | - | - | 是否启用校验和验证 |
| `auto_save_on_scene_change` | true | - | - | 切换场景时是否自动保存 |
| `auto_save_on_event` | true | - | - | 完成重要事件后是否自动保存 |
| `max_save_file_size_mb` | 1 | 0.1 | 10 | 存档文件最大大小限制 |

**参数交互**：
- `compression_enabled` 影响 `save_file_format`：压缩时建议使用binary格式
- `checksum_enabled` 影响性能：启用时保存速度稍慢，但更安全
- `auto_save_interval` 影响游戏流畅度：间隔太短可能导致卡顿

## Visual/Audio Requirements

### 视觉反馈
| 操作 | 视觉效果 | 持续时间 |
|------|----------|----------|
| 自动保存 | 右下角存档图标（软盘图标）淡入淡出 | 1秒 |
| 手动保存 | 保存按钮按下动画 + 成功勾号 | 0.5秒 |
| 加载成功 | 屏幕淡入效果 | 0.5秒 |
| 存档失败 | 屏幕闪烁红色警告 | 1秒 |
| 存档列表 | 存档卡片悬停放大效果 | 即时 |

### 音频反馈
| 操作 | 音效描述 | 音量 |
|------|----------|------|
| 自动保存 | 轻柔"叮"声（不打断游戏） | 30% |
| 手动保存成功 | 短促确认音效（清脆） | 60% |
| 加载成功 | 卷轴展开声 | 70% |
| 存档失败 | 低沉警告音效 | 80% |
| 删除存档 | 纸张撕裂声 | 60% |

### 存档卡片信息
- 槽位名称（可编辑）
- 保存时间（格式：YYYY-MM-DD HH:MM）
- 游戏进度（已完成章节/区域）
- 游戏时长
- 缩略图（可选，高级功能）

### 交互规则
- 点击槽位选中
- 双击槽位直接加载（加载模式）
- 保存模式下点击"保存"按钮
- 删除需要二次确认对话框
- 自动保存槽位不可手动覆盖

## Acceptance Criteria

| 编号 | 验收标准 | 测试方法 |
|------|----------|----------|
| AC-1 | 游戏退出时自动保存当前状态 | 退出游戏后重新启动，验证状态保持 |
| AC-2 | 玩家可手动保存游戏到指定槽位 | 保存后在存档列表中可见 |
| AC-3 | 玩家可加载指定槽位的存档 | 加载后游戏状态与保存时一致 |
| AC-4 | 存档列表显示正确的存档信息 | 验证槽位名称、时间戳、进度信息 |
| AC-5 | 存档文件包含校验和，防止篡改 | 修改存档文件后加载失败 |
| AC-6 | 存档损坏时显示错误信息 | 损坏文件加载时显示友好提示 |
| AC-7 | 存档版本不兼容时尝试迁移 | 旧版本存档加载时数据正确迁移 |
| AC-8 | 存档槽位已满时提示玩家 | 保存到满槽时显示警告 |
| AC-9 | 存档操作不影响游戏流畅度 | 保存/加载时无明显卡顿 |
| AC-10 | 所有系统数据正确保存和加载 | 验证各系统状态完整性 |
