# World State System

> **Status**: In Design
> **Author**: User + Agents
> **Last Updated**: 2026-06-03
> **Implements Pillar**: Pillar 1 (Choice = Rules), Pillar 3 (Explore = Reward)

## Overview

世界状态系统是《命运卡牌局》的世界管理层，负责追踪和管理每个世界/区域的解锁状态、进度和条件。系统记录玩家在各个世界中的进度，根据选择和成就解锁新区域，并管理世界之间的连接关系。

## Player Fantasy

**"我正在逐步揭开这个世界的秘密"**

玩家应该感受到：
- **探索欲**：每个区域都有新的发现等待着我
- **成就感**：解锁新区域是对我的选择的奖励
- **好奇心**：世界之间有隐藏的连接等待发现
- **掌控感**：我的选择决定了哪些区域对我开放
- **期待感**：下一个区域会有什么在等着我

**参考体验**：塞尔达传说的区域解锁 + 空洞骑士的地图探索 + 杀戮尖塔的路径选择

## Detailed Design

### Core Rules

**世界结构**：
- 每个世界有多个区域（3-5个）
- 每个区域有独立的进度和解锁条件
- 区域之间有连接关系（线性/分支/隐藏）

**解锁条件类型**：
- **进度解锁**：完成前置区域
- **选择解锁**：做出特定选择
- **印记解锁**：获得特定印记
- **物品解锁**：拥有特定物品
- **隐藏解锁**：满足特殊条件

**状态管理**：
- **未解锁**：条件未满足
- **已解锁**：条件满足，可进入
- **已探索**：已进入过
- **已完成**：完成所有内容
- **隐藏**：特殊状态，需要发现

### States and Transitions

| 当前状态 | 触发条件 | 目标状态 | 说明 |
|----------|----------|----------|------|
| 未解锁 | 满足解锁条件 | 已解锁 | 解锁新区域 |
| 已解锁 | 玩家进入 | 已探索 | 标记为已探索 |
| 已探索 | 完成所有内容 | 已完成 | 标记为已完成 |
| 隐藏 | 发现隐藏条件 | 已解锁 | 发现隐藏区域 |
| 已完成 | 重新进入 | 已探索 | 允许重复探索 |

### Interactions with Other Systems

| 交互系统 | 方向 | 接口 | 说明 |
|----------|------|------|------|

| 区域系统 | 输出 | get_area_data() | 获取区域数据 |
| 世界探索系统 | 输出 | get_world_map() | 获取世界地图 |

| NPC系统 | 输入 | get_npc_states() | 获取NPC状态 |

## Formulas

**区域解锁公式**：

`area_unlocked = (progress_met) AND (choice_met) AND (mark_met) AND (item_met)`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| progress_met | - | bool | true/false | 是否满足进度要求 |
| choice_met | - | bool | true/false | 是否满足选择要求 |
| mark_met | - | bool | true/false | 是否满足印记要求 |
| item_met | - | bool | true/false | 是否满足物品要求 |

**输出范围**：true/false
**示例**：解锁城堡区域需要完成森林 AND 选择帮助村民 AND 善良印记≥3

**世界完成度公式**：

`world_completion = (explored_areas / total_areas) × 100%`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| explored_areas | - | int | 0-10 | 已探索区域数 |
| total_areas | - | int | 1-10 | 总区域数 |

**输出范围**：0%到100%
**示例**：世界有5个区域，已探索3个 → completion=60%

**隐藏发现公式**：

`hidden_discovered = (special_condition_met) AND (discovery_chance_roll)`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| special_condition_met | - | bool | true/false | 是否满足特殊条件 |
| discovery_chance_roll | - | bool | true/false | 发现概率判定 |

**输出范围**：true/false
**示例**：在特定时间访问特定地点 AND 概率判定通过 → 发现隐藏区域

## Edge Cases

| # | 边界情况 | 处理方式 |
|---|----------|----------|
| 1 | **所有区域已完成** | 显示"世界完成"，解锁隐藏内容 |
| 2 | **解锁条件冲突** | 优先满足最近的条件 |
| 3 | **隐藏区域已发现** | 标记为已解锁，不再隐藏 |
| 4 | **世界状态异常** | 验证状态合法性，修复异常 |
| 5 | **区域重复进入** | 允许重复进入，标记为已探索 |
| 6 | **解锁条件变更** | 重新计算解锁状态 |
| 7 | **世界进度丢失** | 从存档恢复状态 |
| 8 | **区域连接断裂** | 验证连接完整性，修复断裂 |

## Tuning Knobs

| 调优参数 | 默认值 | 安全范围 | 极端行为 |
|----------|--------|----------|----------|
| 区域数量/世界 | 5 | 3-8 | 太少=内容不足，太多=管理复杂 |
| 解锁条件数量 | 2 | 1-4 | 太少=太容易，太多=过于复杂 |
| 隐藏区域比例 | 20% | 10-30% | 太少=无惊喜，太多=过于分散 |
| 世界完成奖励 | 100金币 | 50-500金币 | 太低=无动力，太高=破坏平衡 |
| 区域探索奖励 | 50金币 | 20-200金币 | 太低=无动力，太高=破坏平衡 |
| 隐藏发现概率 | 10% | 5-20% | 太低=不可发现，太高=无惊喜 |
| 世界重置冷却 | 24小时 | 1-72小时 | 太短=无意义，太长=过于限制 |
| 区域连接验证 | 每次进入 | 每次/每天 | 每次=性能开销，每天=可能遗漏 |

## Visual/Audio Requirements

**视觉效果**：
- **世界地图**：手绘风格的世界地图，带区域标记
- **区域解锁**：区域图标发光 + 解锁动画
- **隐藏发现**：特殊光效 + 发现动画
- **世界完成**：完成动画 + 奖励显示
- **区域连接**：连接线发光 + 路径指示

**音频效果**：
- **区域解锁**：解锁音效
- **隐藏发现**：发现音效 + 惊喜音乐
- **世界完成**：完成音效 + 成就音乐
- **区域进入**：区域环境音效
- **地图打开**：地图音效

- 点击区域图标 → 查看区域详情

- 点击隐藏区域（如已发现）→ 进入隐藏区域
- 拖拽地图 → 移动视图

## Acceptance Criteria

| # | 验收标准 | Given-When-Then |
|---|----------|-----------------|
| AC-1 | 区域解锁 | GIVEN 满足解锁条件 WHEN 条件满足 THEN 区域正确解锁 |
| AC-2 | 区域进入 | GIVEN 区域已解锁 WHEN 玩家点击进入 THEN 正确进入区域 |
| AC-3 | 隐藏发现 | GIVEN 满足隐藏条件 WHEN 发现判定通过 THEN 隐藏区域正确显示 |
| AC-4 | 世界完成 | GIVEN 所有区域已完成 WHEN 完成判定 THEN 世界完成动画显示 |
| AC-5 | 状态保存 | GIVEN 世界状态变更 WHEN 保存游戏 THEN 状态正确保存 |
| AC-6 | 状态加载 | GIVEN 加载存档 WHEN 游戏启动 THEN 世界状态正确恢复 |
| AC-7 | 地图显示 | GIVEN 打开世界地图 WHEN 地图加载 THEN 所有区域正确显示 |
| AC-8 | 解锁条件 | GIVEN 查看未解锁区域 WHEN 显示条件 THEN 条件清晰显示 |
| AC-9 | 区域连接 | GIVEN 区域有连接 WHEN 查看地图 THEN 连接线正确显示 |
| AC-10 | 完成度显示 | GIVEN 查看世界完成度 WHEN 显示完成度 THEN 百分比正确计算 |
