# Story Tracking System

> **Status**: Designed
> **Author**: User + Agents
> **Last Updated**: 2026-06-03
> **Implements Pillar**: Pillar 1 (Choice = Rules), Pillar 3 (Explore = Reward)

## Overview

剧情追踪系统是《命运卡牌局》的叙事管理层，负责追踪和管理游戏中的三层剧情结构（主线/支线/隐藏剧情）。系统记录玩家的剧情进度，根据选择触发不同的剧情分支，并管理隐藏剧情的发现条件。

## Player Fantasy

**"我的故事在不断展开，每个选择都在书写新的篇章"**

玩家应该感受到：
- **沉浸感**：故事在不断推进，有明确的叙事节奏
- **掌控感**：我的选择决定了故事的走向
- **惊喜感**：隐藏剧情在特定条件下触发
- **成就感**：完成剧情获得奖励和解锁
- **重玩价值**：不同选择带来不同故事体验

**参考体验**：传说之下的多结局 + 巫师3的支线任务 + 空洞骑士的隐藏剧情

## Detailed Design

### Core Rules

**三层剧情结构**：
1. **主线剧情**：游戏的核心故事，推动游戏进度
2. **支线剧情**：可选的故事内容，丰富世界观
3. **隐藏剧情**：需要特殊条件触发的彩蛋内容

**剧情状态**：
- **未开始**：条件未满足
- **进行中**：剧情已开始，正在推进
- **已完成**：剧情已完成
- **失败**：剧情失败（可重试或永久失败）
- **隐藏**：需要特殊条件发现

**剧情触发条件**：
- **选择触发**：做出特定选择
- **印记触发**：获得特定印记
- **进度触发**：完成前置剧情
- **探索触发**：发现特定区域
- **物品触发**：拥有特定物品

**剧情分支**：
- 每个剧情节点有多个选择
- 选择导向不同的剧情分支
- 分支可能合并或独立发展
- 某些分支有时间限制

### States and Transitions

| 当前状态 | 触发条件 | 目标状态 | 说明 |
|----------|----------|----------|------|
| 未开始 | 满足触发条件 | 进行中 | 开始剧情 |
| 进行中 | 推进剧情 | 进行中 | 继续剧情 |
| 进行中 | 完成剧情 | 已完成 | 结束剧情 |
| 进行中 | 剧情失败 | 失败 | 剧情失败 |
| 失败 | 重试条件满足 | 进行中 | 重新开始 |
| 隐藏 | 发现条件满足 | 进行中 | 发现隐藏剧情 |
| 已完成 | 重新触发 | 进行中 | 重复体验 |

## Formulas

**剧情解锁公式**：

`story_unlocked = (choice_met) AND (mark_met) AND (progress_met) AND (item_met)`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| choice_met | - | bool | true/false | 是否满足选择要求 |
| mark_met | - | bool | true/false | 是否满足印记要求 |
| progress_met | - | bool | true/false | 是否满足进度要求 |
| item_met | - | bool | true/false | 是否满足物品要求 |

**输出范围**：true/false
**示例**：解锁隐藏剧情需要选择帮助村民 AND 善良印记≥5 AND 完成支线A AND 拥有钥匙

**剧情完成度公式**：

`story_completion = (completed_stories / total_stories) × 100%`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| completed_stories | - | int | 0-50 | 已完成剧情数 |
| total_stories | - | int | 10-50 | 总剧情数 |

**输出范围**：0%到100%
**示例**：游戏有30个剧情，已完成18个 → completion=60%

**剧情影响力公式**：

`story_impact = base_value × branch_multiplier × consequence_weight`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| base_value | - | int | 1-10 | 剧情基础价值 |
| branch_multiplier | - | float | 0.5-2.0 | 分支乘数 |
| consequence_weight | - | float | 0.8-1.2 | 后果权重 |

**输出范围**：0.4到24
**示例**：主线剧情base=10，分支乘数1.5，后果权重1.0 → impact=15

## Edge Cases

| # | 边界情况 | 处理方式 |
|---|----------|----------|
| 1 | **剧情条件冲突** | 优先满足最近的条件 |
| 2 | **剧情死锁（无法继续）** | 提供跳过或重试机制 |
| 3 | **隐藏剧情已发现** | 标记为已发现，不再隐藏 |
| 4 | **剧情分支合并** | 合并后的剧情继承分支进度 |
| 5 | **剧情时间限制超时** | 标记为失败，允许重试 |
| 6 | **剧情NPC死亡** | 相关剧情终止，但印记保留 |
| 7 | **剧情状态异常** | 验证状态合法性，修复异常 |
| 8 | **剧情重复触发** | 允许重复体验，保留完成状态 |

## Tuning Knobs

| 调优参数 | 默认值 | 安全范围 | 极端行为 |
|----------|--------|----------|----------|
| 主线剧情数量 | 10 | 5-15 | 太少=内容不足，太多=过于冗长 |
| 支线剧情数量 | 20 | 10-30 | 太少=内容不足，太多=过于分散 |
| 隐藏剧情数量 | 5 | 3-10 | 太少=无惊喜，太多=过于分散 |
| 剧情分支数量 | 3 | 2-5 | 太少=选择受限，太多=过于复杂 |
| 剧情时间限制 | 无限制 | 1小时/1天/无限制 | 限制=增加紧迫感 |
| 剧情重试次数 | 3 | 1-5 | 太少=过于严格，太多=无后果 |
| 剧情奖励倍率 | 1.0 | 0.5-2.0 | 影响剧情奖励 |
| 隐藏发现概率 | 10% | 5-20% | 太低=不可发现，太高=无惊喜 |

## Visual/Audio Requirements

**视觉效果**：
- **剧情界面**：剧情进度条 + 剧情图标
- **剧情触发**：剧情开始动画 + 特效
- **剧情完成**：完成动画 + 奖励显示
- **剧情分支**：分支选择界面
- **隐藏发现**：发现动画 + 惊喜特效
- **剧情回顾**：剧情回顾界面

**音频效果**：
- **剧情开始**：剧情开始音乐
- **剧情进行**：剧情背景音乐
- **剧情完成**：完成音效 + 成就音乐
- **隐藏发现**：发现音效 + 惊喜音乐
- **剧情分支**：选择音效

## UI Requirements

**剧情追踪界面**：
```
┌─────────────────────────────────────────┐
│  剧情类型选择（主线/支线/隐藏）         │
├─────────────────────────────────────────┤
│                                         │
│  剧情列表（图标 + 名称 + 状态）         │
│  ┌─────┐  主线：命运的召唤 [进行中]     │
│  │ 📖  │  支线：村民的请求 [已完成]     │
│  └─────┘  隐藏：失落的记忆 [未开始]     │
│                                         │
├─────────────────────────────────────────┤
│  剧情详情（描述/进度/奖励）             │
├─────────────────────────────────────────┤
│  剧情完成度（进度条 + 百分比）          │
└─────────────────────────────────────────┘
```

**交互规则**：
- 点击剧情类型 → 切换剧情列表
- 点击剧情 → 查看剧情详情
- 点击进行中剧情 → 继续剧情
- 点击已完成剧情 → 回顾剧情
- 拖拽列表 → 滚动列表

## Acceptance Criteria

| # | 验收标准 | Given-When-Then |
|---|----------|-----------------|
| AC-1 | 剧情触发 | GIVEN 满足触发条件 WHEN 条件满足 THEN 剧情正确开始 |
| AC-2 | 剧情推进 | GIVEN 剧情进行中 WHEN 推进剧情 THEN 剧情正确推进 |
| AC-3 | 剧情完成 | GIVEN 剧情完成条件满足 WHEN 完成判定 THEN 剧情正确完成 |
| AC-4 | 剧情分支 | GIVEN 到达分支点 WHEN 做出选择 THEN 分支正确应用 |
| AC-5 | 隐藏发现 | GIVEN 满足隐藏条件 WHEN 发现判定通过 THEN 隐藏剧情正确显示 |
| AC-6 | 剧情失败 | GIVEN 剧情失败条件满足 WHEN 失败判定 THEN 剧情正确标记为失败 |
| AC-7 | 剧情重试 | GIVEN 剧情失败 WHEN 重试条件满足 THEN 剧情正确重新开始 |
| AC-8 | 状态保存 | GIVEN 剧情状态变更 WHEN 保存游戏 THEN 状态正确保存 |
| AC-9 | 状态加载 | GIVEN 加载存档 WHEN 游戏启动 THEN 剧情状态正确恢复 |
| AC-10 | 剧情界面 | GIVEN 打开剧情界面 WHEN 界面加载 THEN 所有剧情正确显示 |
