# 状态效果系统

> **Status**: In Design
> **Author**: [user + agents]
> **Last Updated**: 2026-06-03
> **Implements Pillar**: 支柱2 — 爽快的策略深度

## Overview

状态效果系统是战斗系统的核心组件，负责管理所有Buff（增益）和Debuff（减益）效果的生命周期。它定义了效果的类型、持续时间、叠加规则和触发时机，为伤害计算系统、召唤物系统和卡牌战斗系统提供效果数据。玩家通过卡牌触发效果，感受到"我的策略组合产生了强大的连锁反应"——一张增益卡牌配合攻击卡牌，可以打出毁灭性的伤害。

## Player Fantasy

**状态效果系统**是玩家间接体验的系统——玩家通过卡牌触发效果，但不直接操作系统。

**情感目标**："我的策略组合产生了强大的连锁反应"

**玩家应该感受到**：
- **策略深度**：我可以通过增益/减益卡牌组合来打出更高伤害
- **掌控感**：我选择的卡牌组合直接影响战斗结果
- **爽快感**：看到增益效果叠加后的伤害数字爆发

**情感锚点**：像炉石传说中一张增益卡牌配合随从打出超高伤害的爽快感——"我的Combo打出来了！"

## Detailed Design

### Core Rules

1. **效果类型**：
   - **Buff（增益）**：增强角色能力（如攻击力+20%、暴击率+10%）
   - **Debuff（减益）**：削弱角色能力（如防御力-30%、速度-50%）
   - **DoT（持续伤害）**：每回合造成固定伤害（如中毒、灼烧）
   - **HoT（持续治疗）**：每回合恢复固定生命（如再生）

2. **效果属性**：
   - **效果ID**：唯一标识符
   - **效果名称**：显示名称
   - **效果类型**：Buff/Debuff/DoT/HoT
   - **效果值**：数值（如+20%、-30%、每回合5点伤害）
   - **持续时间**：回合数（1-10回合）
   - **叠加规则**：是否可叠加、叠加方式（刷新/叠加/替换）

3. **触发时机**：
   - **回合开始**：DoT/HoT效果触发
   - **回合结束**：持续时间减少，效果移除
   - **卡牌打出时**：Buff/Debuff效果应用
   - **受到伤害时**：防御类Buff生效

4. **叠加规则**：
   - **同类效果**：取最高值（如两个攻击力+20%和+30%，取+30%）
   - **不同类效果**：独立计算（如攻击力+20%和暴击率+10%同时生效）
   - **刷新机制**：新效果刷新持续时间，不改变效果值

### States and Transitions

| 状态 | 描述 | 触发条件 |
|------|------|----------|
| **待应用** | 效果等待应用 | 卡牌打出 |
| **生效中** | 效果正在生效 | 效果应用到目标 |
| **即将过期** | 效果剩余1回合 | 回合结束检查 |
| **已过期** | 效果已移除 | 持续时间结束 |
| **被净化** | 效果被主动移除 | 净化技能/卡牌 |

**状态转换规则**：
- 待应用 → 生效中：卡牌效果应用
- 生效中 → 即将过期：剩余1回合
- 即将过期 → 已过期：回合结束
- 生效中 → 被净化：净化技能/卡牌触发
- 已过期/被净化 → 移除：从效果列表中删除

### Interactions with Other Systems

| 系统 | 数据流向 | 接口描述 |
|------|----------|----------|
| **卡牌数据库** | 输入 | 读取卡牌效果定义（效果类型、值、持续时间） |
| **伤害计算系统** | 输出 | 提供Buff/Debuff对伤害的加成/削弱值 |
| **召唤物系统** | 输出 | 提供召唤物身上的效果列表 |
| **卡牌战斗系统** | 输出 | 提供当前角色身上的效果列表 |
| **角色属性系统** | 输出 | 提供效果对属性的临时修改 |

## Formulas

### 效果值计算公式

`effect_value = base_value × (1 + buff_multiplier)`

**变量：**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_value | V_b | int | 1-50 | 效果基础值（来自CardEffect.value） |
| buff_multiplier | M_b | float | 0.0-2.0 | Buff/Debuff倍率（0.0=无效果，1.0=双倍，2.0=三倍） |
| effect_value | V_e | int | 1-150 | 最终效果值 |

**输出范围：** 1到150（正常游戏范围）
**示例：** 基础值10，Buff倍率0.5，最终效果值 = 10 × (1 + 0.5) = 15

### DoT/HoT伤害/治疗公式

`dot_hot_value = base_value × (1 + duration_modifier)`

**变量：**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_value | V_b | int | 1-20 | 每回合基础伤害/治疗值 |
| duration_modifier | M_d | float | 0.0-1.0 | 持续时间修正（持续时间越长，每回合效果越弱） |
| dot_hot_value | V_d | int | 1-30 | 每回合实际伤害/治疗值 |

**输出范围：** 1到30（正常游戏范围）
**示例：** 基础值10，持续时间3回合，duration_modifier=0.2，每回合效果值 = 10 × (1 + 0.2) = 12

### 效果叠加公式

`final_effect = max(effects_of_same_type)`

**变量：**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| effects_of_same_type | E_list | Array[int] | 1-150 | 同类型效果值列表 |
| final_effect | E_f | int | 1-150 | 最终生效的效果值 |

**输出范围：** 1到150（取最高值）
**示例：** 两个攻击力+20%和+30%，最终效果值 = max(20, 30) = 30

## Edge Cases

- **如果效果值为0**：效果不应用，显示"无效效果"提示。
- **如果效果值超过上限（150）**：效果值限制为150，防止数值溢出。
- **如果持续时间为0**：效果立即过期，不应用。
- **如果持续时间为负数**：效果不应用，显示"无效持续时间"提示。
- **如果同时有多个同类效果**：取最高值，其他效果不生效但保留在列表中。
- **如果效果被净化时正在生效**：效果立即移除，不触发过期效果。
- **如果角色死亡时有效果**：所有效果立即移除，不触发DoT/HoT。
- **如果效果叠加导致属性溢出**：属性值限制在0-149范围内。

## Tuning Knobs

| 调优参数 | 默认值 | 范围 | 说明 |
|----------|--------|------|------|
| **最大效果值** | 150 | 50-300 | 单个效果的最大值 |
| **最大持续时间** | 10 | 1-20 | 效果的最大持续回合数 |
| **DoT/HoT基础值** | 5 | 1-20 | 每回合基础伤害/治疗值 |
| **DoT/HoT持续时间修正** | 0.2 | 0.0-1.0 | 持续时间越长，每回合效果越弱 |
| **效果叠加规则** | 取最高值 | 取最高值/叠加/替换 | 同类效果的叠加方式 |
| **净化效果范围** | 单个 | 单个/全体/随机 | 净化技能移除效果的范围 |
| **效果刷新机制** | 刷新持续时间 | 刷新/不刷新 | 新效果是否刷新持续时间 |
| **效果显示时间** | 2秒 | 1-5秒 | 效果应用时的UI显示时间 |

## Visual/Audio Requirements

### 视觉效果
- **Buff效果**：绿色光圈/上升箭头图标
- **Debuff效果**：红色光圈/下降箭头图标
- **DoT效果**：紫色毒雾/火焰特效
- **HoT效果**：绿色光点/生命恢复特效
- **效果应用**：目标周围闪烁对应颜色光效
- **效果移除**：光效消散动画

### 音频效果
- **Buff应用**：清脆的增益音效（如叮声）
- **Debuff应用**：低沉的减益音效（如嗡声）
- **DoT触发**：每回合毒伤/灼烧音效
- **HoT触发**：每回合治疗音效
- **效果净化**：净化音效（如清脆的破碎声）
- **效果过期**：效果消散音效

## UI Requirements

### 效果图标显示
- **位置**：角色头像下方或生命条旁边
- **大小**：24x24像素
- **排列**：水平排列，最多显示6个效果
- **悬停提示**：鼠标悬停显示效果名称、值、剩余回合数

### 效果详情面板
- **显示内容**：效果名称、类型、值、持续时间、叠加层数
- **布局**：垂直列表，按类型分组（Buff/Debuff/DoT/HoT）
- **交互**：点击可查看效果来源（来自哪张卡牌）

### 效果应用提示
- **位置**：目标上方
- **动画**：向上漂浮，1秒后消失
- **内容**：效果名称和值（如"攻击力+20%"）
- **颜色**：Buff绿色，Debuff红色，DoT紫色，HoT绿色

## Acceptance Criteria

| 编号 | 验收标准 | 测试方法 |
|------|----------|----------|
| AC-1 | Buff效果正确增强角色能力 | 应用Buff后验证属性值变化 |
| AC-2 | Debuff效果正确削弱角色能力 | 应用Debuff后验证属性值变化 |
| AC-3 | DoT效果每回合造成正确伤害 | 验证每回合伤害值 |
| AC-4 | HoT效果每回合恢复正确生命 | 验证每回合治疗值 |
| AC-5 | 效果持续时间正确递减 | 验证每回合持续时间减少 |
| AC-6 | 效果过期后正确移除 | 验证效果在持续时间结束时移除 |
| AC-7 | 同类效果正确叠加（取最高值） | 应用多个同类效果，验证取最高值 |
| AC-8 | 净化效果正确移除目标效果 | 使用净化技能，验证效果移除 |
| AC-9 | 效果图标正确显示在UI上 | 验证效果图标位置和内容 |
| AC-10 | 效果应用时显示正确提示 | 验证效果应用提示显示 |
