# Card Deck Building System

> **Status**: Designed
> **Author**: User + Agents
> **Last Updated**: 2026-06-03
> **Implements Pillar**: Pillar 2 (Satisfying Depth), Pillar 4 (Persistent Growth)

## Overview

卡组构筑系统是《命运卡牌局》的卡牌管理层，负责管理玩家的卡组构建规则和限制。系统定义卡组的大小限制、卡牌类型限制、卡牌数量限制，并提供卡组优化建议。

## Player Fantasy

**"我在构建一副独一无二的卡组"**

玩家应该感受到：
- **创造力**：我可以自由组合卡牌
- **策略性**：卡组构建需要深思熟虑
- **成就感**：构建出强力卡组的满足感
- **个性化**：我的卡组反映了我的游戏风格
- **成长感**：卡组在不断优化和进化

**参考体验**：杀戮尖塔的卡组构筑 + 游戏王的卡组构建 + 炉石传说的套牌构筑

## Detailed Design

### Core Rules

**卡组规则**：
- **卡组大小**：最小20张，最大40张
- **卡牌数量限制**：每张卡牌最多携带3张
- **卡牌类型限制**：无限制（可混合所有类型）
- **元素限制**：无限制（可混合所有元素）

**卡组结构**：
- **主卡组**：用于战斗的卡牌（20-40张）
- **备选卡组**：可替换的卡牌（0-10张）
- **收藏**：所有已获得的卡牌

**卡组操作**：
- **添加卡牌**：从收藏添加卡牌到卡组
- **移除卡牌**：从卡组移除卡牌到收藏
- **替换卡牌**：用备选卡牌替换主卡组卡牌
- **清空卡组**：清空主卡组所有卡牌
- **保存卡组**：保存当前卡组配置
- **加载卡组**：加载已保存的卡组配置

**卡组验证**：
- 卡组大小必须在限制范围内
- 每张卡牌数量不能超过限制
- 卡组必须包含至少1张攻击卡牌

### States and Transitions

| 当前状态 | 触发条件 | 目标状态 | 说明 |
|----------|----------|----------|------|

| 构建中 | 清空卡组 | 空卡组 | 清空卡组 |

## Formulas

**卡组强度公式**：

`deck_strength = (card_quality × card_synergy × card_diversity) / deck_size`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| card_quality | - | float | 0-100 | 卡牌质量平均值 |
| card_synergy | - | float | 0-2 | 卡牌协同系数 |
| card_diversity | - | float | 0.5-1.5 | 卡牌多样性系数 |
| deck_size | - | int | 20-40 | 卡组大小 |

**输出范围**：0到15
**示例**：card_quality=60，card_synergy=1.5，card_diversity=1.2，deck_size=30 → strength=3.6

**卡组平衡公式**：

`deck_balance = (attack_ratio × defense_ratio × utility_ratio) ^ (1/3)`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| attack_ratio | - | float | 0-1 | 攻击卡牌比例 |
| defense_ratio | - | float | 0-1 | 防御卡牌比例 |
| utility_ratio | - | float | 0-1 | 辅助卡牌比例 |

**输出范围**：0到1
**示例**：attack_ratio=0.4，defense_ratio=0.3，utility_ratio=0.3 → balance=0.33

**卡组优化建议公式**：

`optimization_score = (missing_types × 0.3) + (weak_synergy × 0.4) + (size_imbalance × 0.3)`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| missing_types | - | float | 0-1 | 缺失类型比例 |
| weak_synergy | - | float | 0-1 | 弱协同比例 |
| size_imbalance | - | float | 0-1 | 大小不平衡比例 |

**输出范围**：0到1
**示例**：missing_types=0.2，weak_synergy=0.3，size_imbalance=0.1 → score=0.21

## Edge Cases

| # | 边界情况 | 处理方式 |
|---|----------|----------|
| 1 | **卡组为空** | 提示"卡组为空"，阻止进入战斗 |
| 2 | **卡组超过上限** | 提示"卡组已满"，阻止添加卡牌 |
| 3 | **卡牌数量超限** | 提示"卡牌数量超限"，阻止添加 |
| 4 | **卡组无攻击卡** | 提示"缺少攻击卡"，警告但允许 |
| 5 | **卡组配置丢失** | 从存档恢复配置 |
| 6 | **卡牌被删除** | 从卡组移除已删除卡牌 |
| 7 | **卡牌升级后** | 卡组中的卡牌自动升级 |
| 8 | **卡组重复保存** | 覆盖已保存配置 |

## Tuning Knobs

| 调优参数 | 默认值 | 安全范围 | 极端行为 |
|----------|--------|----------|----------|
| 卡组最小大小 | 20 | 15-25 | 太小=策略不足，太大=管理复杂 |
| 卡组最大大小 | 40 | 30-50 | 太小=选择受限，太大=过于复杂 |
| 卡牌数量限制 | 3 | 1-5 | 太少=策略受限，太多=过于重复 |
| 备选卡组大小 | 10 | 5-15 | 太少=替换受限，太多=管理复杂 |
| 卡组保存数量 | 5 | 3-10 | 太少=配置受限，太多=管理复杂 |
| 卡组强度上限 | 15 | 10-20 | 太低=成长受限，太高=过于强大 |
| 卡组平衡权重 | 0.33 | 0.2-0.5 | 影响卡组平衡计算 |
| 优化建议阈值 | 0.3 | 0.2-0.5 | 影响优化建议触发 |

| # | 验收标准 | Given-When-Then |
|---|----------|-----------------|
| AC-1 | 卡组加载 | GIVEN 打开卡组界面 WHEN 界面加载 THEN 卡组正确显示 |
| AC-2 | 卡牌添加 | GIVEN 选择卡牌 WHEN 添加到卡组 THEN 卡牌正确添加 |

| AC-6 | 卡组加载 | GIVEN 点击加载按钮 WHEN 加载执行 THEN 卡组配置正确加载 |
