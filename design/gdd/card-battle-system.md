# Card Battle System

> **Status**: In Design
> **Author**: User + Agents
> **Last Updated**: 2026-06-03
> **Implements Pillar**: Pillar 1 (Choice = Rules), Pillar 2 (Satisfying Depth)

## Overview

卡牌战斗系统是《命运卡牌局》的核心玩法系统，负责管理玩家与敌人之间的卡牌对战。玩家从手牌中选择卡牌打出，系统根据卡牌类型（召唤/直接伤害/环境阵地/增益减益）执行效果，结合伤害计算、状态效果、Combo连锁、环境系统和元素系统，产生最终战斗结果。战斗采用回合制，每回合玩家有能量点用于打出卡牌，目标是将敌方生命值降至0。系统支持SL回档（保存/加载），允许玩家在战斗中重新尝试。

## Player Fantasy

**"我的每一个选择都在战场上产生了有意义的后果"**

玩家应该感受到：
- **掌控感**：我打出的每张卡牌都在改变战场局势
- **爽快感**：Combo连锁触发时，华丽的特效和巨大的伤害数字让我血脉喷张
- **策略深度**：我需要思考能量分配、元素克制、召唤物位置、环境变化
- **惊喜感**：故事印记和规则改写让每场战斗都不一样
- **安全感**：SL回档让我敢于尝试大胆的策略，不怕失败

**参考体验**：杀戮尖塔的卡牌构筑深度 + 游戏王的召唤仪式感 + 炉石传说的即时反馈爽感

## Detailed Design

### Core Rules

**战斗流程**：
1. 战斗开始 → 双方初始化（生命值、能量、手牌）
2. 玩家回合开始 → 抽牌、能量恢复
3. 玩家行动阶段 → 打出卡牌、使用技能、结束回合
4. 敌方回合 → AI执行行动
5. 回合结束 → 处理持续效果、环境变化
6. 胜负判定 → 一方生命值≤0

**能量系统**：
- 每回合初始能量：3点（可通过规则改写改变）
- 每张卡牌有能量消耗（0-10点）
- 能量不足时无法打出卡牌
- 回合结束时剩余能量清零（除非有规则改写保留）

**手牌管理**：
- 初始手牌：5张
- 每回合抽牌：2张（可通过规则改写改变）
- 手牌上限：10张
- 超出上限时，新抽的牌被弃置

**卡牌打出规则**：
- 选择卡牌 → 选择目标（如需要） → 确认打出 → 执行效果
- 召唤类：召唤生物到场上（最多3个位置）
- 直接伤害类：对敌方造成伤害（走伤害计算公式）
- 环境阵地类：改变战场环境（影响所有单位）
- 增益减益类：给己方/敌方添加Buff/Debuff

## Formulas

**能量消耗公式**：

`energy_cost = card_base_cost`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| card_base_cost | - | int | 0-10 | 卡牌基础能量消耗 |

**输出范围**：1到5；0表示免费卡牌
**示例**：一张火球术卡牌 base_cost=3，玩家需要3点能量

**抽牌数量公式**：

`draw_count = base_draw + rule_modifier + mark_modifier`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| base_draw | - | int | 2 | 每回合基础抽牌 |
| rule_modifier | - | int | -1~+2 | 规则改写加成 |
| mark_modifier | - | int | 0~+1 | 故事印记加成 |

**输出范围**：1到5张
**示例**：base_draw=2，rule_modifier=+1，mark_modifier=0 → 抽3张

**Combo触发公式**：

`combo_triggered = (combo_cards_played >= combo_threshold) AND (combo_elements_match)`

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| combo_cards_played | - | int | 1-10 | 本回合打出的卡牌数 |
| combo_threshold | - | int | 2-5 | 触发Combo所需的卡牌数 |
| combo_elements_match | - | bool | true/false | 元素是否匹配 |

**输出范围**：true/false
**示例**：打出3张火系卡牌，threshold=3 → 触发火系Combo

**战斗胜率估算**：

`win_rate ≈ (player_total_damage / enemy_hp) * (1 - enemy_dps / player_hp)`

**输出范围**：0到1（0%=必败，100%=必胜）
**示例**：player_total_damage=50, enemy_hp=100, enemy_dps=10, player_hp=80 → win_rate ≈ 0.5 * 0.875 = 0.4375

## Edge Cases

| # | 边界情况 | 处理方式 |
|---|----------|----------|
| 1 | **能量为0，手牌全高于0消耗** | 允许结束回合，不能打出任何卡牌 |
| 2 | **手牌已满（10张），需要抽牌** | 新抽的牌直接弃置，不进入手牌 |
| 3 | **场上已有3个召唤物，再召唤** | 提示"场地已满"，无法打出召唤卡 |
| 4 | **敌方和玩家同时HP≤0** | 玩家优先判定为失败（保护机制） |
| 5 | **Combo卡牌被打断（被控制）** | Combo计数重置为0 |
| 6 | **环境被多次改变** | 后来的环境覆盖前面的，不叠加 |
| 7 | **规则改写导致能量为负** | 能量最低为0，不能为负数 |
| 8 | **SL回档后敌方AI行为不同** | 敌方AI使用固定种子，回档后行为一致 |

## Tuning Knobs

| 调优参数 | 默认值 | 安全范围 | 极端行为 |
|----------|--------|----------|----------|
| 初始能量 | 3 | 2-5 | 太低=无法出牌，太高=无策略 |
| 每回合抽牌 | 2 | 1-4 | 太少=手牌枯萎，太多=手牌溢出 |
| 手牌上限 | 10 | 7-15 | 太小=选择受限，太大=信息过载 |
| 召唤物上限 | 3 | 2-5 | 太少=召唤无意义，太多=场面混乱 |
| Combo阈值 | 3 | 2-5 | 太低=Combo太容易，太高=Combo不可达 |
| 能量恢复规则 | 清零 | 保留50%/不清零 | 保留=策略变化大 |
| 敌方AI难度 | 普通 | 简单/普通/困难 | 影响敌方决策质量 |
| SL回档限制 | 无限 | 1-3次/战斗 | 限制=增加紧张感 |

## Visual/Audio Requirements

**视觉效果**：
- **卡牌打出**：卡牌飞向目标，带发光轨迹
- **伤害数字**：白色普通伤害，金色暴击，红色克制，蓝色被克制
- **Combo触发**：屏幕震动 + 特效爆发 + "COMBO!"文字
- **召唤物出现**：召唤阵发光 + 单位淡入
- **环境变化**：背景渐变 + 粒子效果
- **Buff/Debuff**：目标身上图标闪烁
- **胜利/失败**：屏幕渐变 + 结算界面

**音频效果**：
- **卡牌打出**：卡牌飞出音效
- **伤害命中**：撞击音效（暴击更响）
- **Combo触发**：激昂音效 + 语音
- **召唤**：召唤阵音效
- **环境变化**：环境音效渐变
- **胜利**：胜利音乐
- **失败**：失败音乐

```

│  敌方信息栏（HP/状态/元素）              │
├─────────────────────────────────────────┤
│                                         │
│  战场区域（召唤物/环境效果）              │
│                                         │
├─────────────────────────────────────────┤
│  玩家信息栏（HP/能量/状态）              │
```

**交互规则**：
- 鼠标点击卡牌 → 选中卡牌
- 鼠标点击目标 → 打出卡牌

- 右键点击卡牌 → 查看详情
- 键盘快捷键：1-9选择手牌，Space结束回合

## Acceptance Criteria

| # | 验收标准 | Given-When-Then |
|---|----------|-----------------|
| AC-1 | 战斗初始化 | GIVEN 玩家进入战斗场景 WHEN 战斗开始 THEN 双方HP、能量、手牌正确初始化 |
| AC-2 | 卡牌打出 | GIVEN 玩家有足够能量 WHEN 选择卡牌并确认 THEN 卡牌效果正确执行，能量扣除 |
| AC-3 | 能量不足 | GIVEN 玩家能量不足 WHEN 尝试打出卡牌 THEN 提示"能量不足"，卡牌无法打出 |
| AC-4 | 伤害计算 | GIVEN 打出伤害卡牌 WHEN 执行效果 THEN 伤害值按公式计算并显示 |
| AC-5 | Combo触发 | GIVEN 满足Combo条件 WHEN 打出Combo卡牌 THEN Combo效果触发，特效显示 |
| AC-6 | 召唤物管理 | GIVEN 场上有空位 WHEN 打出召唤卡 THEN 召唤物出现在指定位置 |
| AC-7 | 环境变化 | GIVEN 打出环境卡 WHEN 效果执行 THEN 战场环境改变，影响所有单位 |
| AC-8 | 胜利判定 | GIVEN 敌方HP≤0 WHEN 回合结束 THEN 战斗胜利，发放奖励 |
| AC-9 | 失败判定 | GIVEN 玩家HP≤0 WHEN 回合结束 THEN 战斗失败，提示SL回档 |
| AC-10 | SL回档 | GIVEN 战斗中 WHEN 玩家选择SL回档 THEN 战斗状态恢复到保存点 |
