# 角色属性系统

> **Status**: In Design
> **Author**: user + agents
> **Last Updated**: 2026-06-03
> **Implements Pillar**: 支柱4 — 持久的成长感

## Overview

角色属性系统是游戏的RPG基础层，定义玩家角色的核心属性、成长曲线和属性计算规则。该系统使用Godot Resource系统存储属性数据，为伤害计算、技能树、卡牌升级等下游系统提供属性查询接口。玩家通过战斗胜利、完成任务、使用道具等方式获得经验值和属性提升，感受到角色在不断变强。

## Player Fantasy

"我的角色在不断变强" — 玩家应该感受到：
- **期待感**：每次升级都期待新的属性提升
- **成就感**：看到数值增长，感受到自己的努力有回报
- **掌控感**：可以选择如何分配属性点，塑造自己的角色风格

**情感锚点**：像RPG游戏中的升级时刻——叮！你的角色变强了！

## Detailed Design

### Core Rules

1. **基础属性**：角色拥有6个基础属性
   - **力量 (STR)**：影响物理伤害和负重
   - **敏捷 (DEX)**：影响暴击率和闪避
   - **智力 (INT)**：影响法术伤害和法力值
   - **体质 (CON)**：影响生命值和防御
   - **感知 (PER)**：影响命中率和发现隐藏
   - **幸运 (LCK)**：影响掉落率和暴击伤害

2. **属性计算**：
   - 基础值 = 种族基础 + 职业基础 + 属性点分配
   - 最终值 = 基础值 + 装备加成 + Buff加成 + 等级加成
   - 属性点来源：每升1级获得3点可分配属性点

3. **等级系统**：
   - 等级范围：1-100
   - 经验值需求：`exp_required = 100 * level^1.5`
   - 等级提升：自动获得基础属性提升 + 可分配属性点

4. **属性上限**：
   - 单属性上限：99（基础）+ 50（加成）= 149
   - 总属性点上限：600（基础）+ 300（加成）= 900

### States and Transitions

| 状态 | 描述 | 触发条件 |
|------|------|----------|
| **创建** | 角色初始属性分配 | 新游戏开始 |
| **成长** | 属性通过升级/装备提升 | 获得经验值/装备 |
| **衰减** | 属性临时降低（Debuff） | 状态效果触发 |
| **重置** | 属性点重新分配 | 使用重置道具 |

**状态转换规则**：
- 创建 → 成长：完成新手教程
- 成长 → 衰减：受到Debuff效果
- 衰减 → 成长：Debuff效果结束
- 成长 → 重置：使用属性重置卷轴
- 重置 → 成长：完成属性重新分配

### Interactions with Other Systems

| 系统 | 数据流向 | 接口描述 |
|------|----------|----------|
| **伤害计算系统** | 输出 | 提供属性值用于伤害公式计算 |
| **技能树系统** | 双向 | 提供等级和属性用于技能解锁；技能提供属性加成 |
| **卡牌升级系统** | 输出 | 提供等级用于卡牌升级条件检查 |
| **RPG成长系统** | 双向 | 提供属性数据用于成长计算；成长系统触发属性提升 |
| **存档系统** | 双向 | 保存/加载角色属性数据 |
| **UI系统** | 输出 | 提供属性数据用于界面显示 |

## Formulas

### 经验值需求公式

`exp_required = 100 * level^1.5`

**变量:**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| 等级 | level | int | 1-100 | 当前角色等级 |

**输出范围:** 100 (等级1) 到 100,000 (等级100)
**示例:** 等级10需要 `100 * 10^1.5 = 3,162` 经验值

### 属性点分配公式

`total_points = 3 * (level - 1)`

**变量:**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| 等级 | level | int | 1-100 | 当前角色等级 |

**输出范围:** 0 (等级1) 到 297 (等级100)
**示例:** 等级10有 `3 * (10-1) = 27` 可分配属性点

### 最终属性值公式

`final_value = base_value + equipment_bonus + buff_bonus + level_bonus`

**变量:**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| 基础值 | base_value | int | 1-99 | 种族+职业+分配点 |
| 装备加成 | equipment_bonus | int | 0-50 | 装备提供的属性加成 |
| Buff加成 | buff_bonus | int | -50 to +50 | 状态效果提供的临时加成 |
| 等级加成 | level_bonus | int | 0-100 | 等级自动提供的属性 |

**输出范围:** 0 (最低) 到 299 (最高)
**示例:** 基础50 + 装备20 + Buff10 + 等级30 = 110

## Edge Cases

- **如果属性值溢出上限**：属性值被限制在最大值（149），不会超过。显示警告提示"属性已达到上限"。
- **如果属性值低于下限**：属性值被限制在最小值（0），不会低于0。显示警告提示"属性已达到下限"。
- **如果多个Buff同时影响同一属性**：所有Buff加成叠加计算，但最终值受上限限制。
- **如果装备加成导致属性溢出**：装备加成正常计算，但最终值受上限限制。提示"部分装备加成未生效"。
- **如果属性重置时属性点不足**：重置失败，提示"没有足够的属性点进行重置"。
- **如果等级提升时属性点溢出**：超出上限的属性点被保留，不会丢失。提示"你有X个未分配的属性点"。
- **如果同时有正面和负面Buff**：正负Buff相互抵消计算，最终值可能为负（但受下限限制）。

## Tuning Knobs

| 参数 | 默认值 | 范围 | 描述 | 极端行为 |
|------|--------|------|------|----------|
| **每级属性点** | 3 | 1-5 | 每升1级获得的可分配属性点 | 1点=成长缓慢，5点=快速成长 |
| **属性上限** | 149 | 100-200 | 单属性最大值 | 100=限制深度，200=过度强大 |
| **经验需求指数** | 1.5 | 1.0-2.0 | 经验值曲线的指数 | 1.0=线性，2.0=指数增长 |
| **基础属性值** | 10 | 5-20 | 种族/职业的基础属性 | 5=弱小起点，20=强大起点 |
| **等级属性加成** | 1 | 0-2 | 等级自动提供的属性 | 0=纯分配，2=快速成长 |
| **Buff叠加上限** | 5 | 1-10 | 同时生效的Buff数量 | 1=限制策略，10=复杂组合 |
| **属性重置成本** | 1000 | 500-5000 | 重置属性点的金币消耗 | 500=易重置，5000=谨慎选择 |
| **经验获取倍率** | 1.0 | 0.5-2.0 | 战斗获得经验的倍率 | 0.5=慢节奏，2.0=快节奏 |

### Art Bible 对齐

- **视觉风格**: 华丽的暗黑童话 — 属性界面应使用深色背景+金色边框
- **色彩系统**: 金色=属性提升，红色=属性降低，蓝色=可分配点
- **UI风格**: 卡牌风格的边框和装饰元素

## UI Requirements

### 角色属性界面

**布局**：
- 左侧：角色立绘和等级显示
- 中间：6个属性条（力量/敏捷/智力/体质/感知/幸运）
- 右侧：可分配属性点和分配按钮
- 底部：属性说明和计算公式预览

**交互规则**：
- 点击属性条：显示详细信息（基础值/加成/最终值）
- 点击 +/- 按钮：分配/回收属性点
- 长按属性条：显示属性影响的系统列表
- 拖拽属性条：快速分配多个属性点

### 属性显示格式

```
力量: 45/149 (+12装备 +5Buff +28等级)
├── 基础值: 45
├── 装备加成: +12
├── Buff加成: +5
├── 等级加成: +28
└── 最终值: 90
```

### 响应式设计

- **PC**: 完整界面，所有信息可见
- **手柄**: 支持D-pad导航，按钮映射
- **最小分辨率**: 1280x720

## Acceptance Criteria

| 编号 | 验收标准 | 测试方法 |
|------|----------|----------|
| AC-1 | 角色创建时可分配初始属性点 | 创建新角色，验证属性分配界面 |
| AC-2 | 升级时获得正确的属性点 | 升级角色，验证属性点增加 |
| AC-3 | 属性值计算正确（基础+装备+Buff+等级） | 配置不同组合，验证最终值 |
| AC-4 | 属性值不超过上限（149） | 尝试超过上限，验证限制生效 |
| AC-5 | 属性值不低于下限（0） | 尝试低于下限，验证限制生效 |
| AC-6 | 多个Buff正确叠加 | 添加多个Buff，验证叠加计算 |
| AC-7 | 属性重置功能正常 | 使用重置卷轴，验证属性点返还 |
| AC-8 | 属性界面正确显示所有信息 | 打开界面，验证数据显示 |
| AC-9 | 属性变化有正确的视觉/音频反馈 | 改变属性，验证特效和音效 |
| AC-10 | 属性数据正确保存和加载 | 保存游戏，重新加载，验证数据一致 |
