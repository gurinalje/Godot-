## Section 1: 视觉身份声明

### 一句话视觉规则

> **"华丽的暗黑童话"** — 每一帧都像一幅精心装裱的塔罗牌，黑暗中透出命运的光芒。

### 支撑原则

**原则1：装饰性边框即世界观（服务于"选择即规则"）**

> 当视觉元素有歧义时，选择有装饰性边框的设计。边框的复杂度反映选择的重要性——普通选择用简单边框，重大选择用华丽边框。

**设计测试**：如果一个视觉元素没有边框或边框过于简单，它可能不够重要。

---

**原则2：色彩对比即反馈（服务于"爽快的策略深度"）**

> 当视觉反馈有歧时，选择高对比度的色彩方案。暗色背景 + 亮色技能特效 = 即时满足感。

**设计测试**：如果一个技能特效在视觉上不够"爽"，它可能需要更强烈的色彩对比。

---

**原则3：符号化构图即发现（服务于"探索奖励好奇心"）**

> 当场景构图有歧义时，选择有隐藏符号的设计。玩家探索时应该能发现视觉线索，暗示隐藏内容。

**设计测试**：如果一个隐藏区域在视觉上没有任何暗示，玩家不会想探索它。

---

**原则4：成长可视化即进度（服务于"持久的成长感"）**

> 当角色/卡组成长有歧义时，选择可见的视觉进化。卡牌升级、角色外观变化、世界色彩变化都应该反映进度。

**设计测试**：如果玩家玩了1小时后视觉上没有变化，他们感觉不到成长。

## Section 2: 情绪与氛围

### 工具约束

> 本游戏使用像素艺术风格（Aseprite），所有视觉效果需在像素艺术框架内实现。

### 游戏状态情绪目标

| 状态 | 主要情绪 | 灯光特征 | 氛围词 | 能量 | 像素视觉元素 |
|------|----------|----------|--------|------|--------------|
| **探索** | 好奇、神秘、期待 | 冷色调为主，暖色点缀 | 神秘、静谧、暗藏玄机 | 平稳 | 环境中的发光像素点、可互动的闪烁物体 |
| **对话** | 紧张、期待、情感连接 | 聚焦光源，突出NPC | 亲密、戏剧性、张力 | 沉思 | NPC身上的装饰性像素边框 |
| **战斗准备** | 紧张、策略、蓄势待发 | 冷色主调，暖色点缀 | 肃杀、专注、仪式感 | 平稳 | 卡牌悬浮的像素动画 |
| **卡牌对战** | 兴奋、爽快、掌控感 | 高对比度，技能特效爆裂 | 狂热、激烈、华丽 | 狂热 | 技能释放的像素粒子、Combo连锁反馈 |
| **胜利** | 成就、释放、喜悦 | 金色暖光，从暗到明 | 辉煌、荣耀、释放 | 高涨 | 金色像素边框、胜利动画 |
| **失败** | 挫败但不绝望、反思 | 冷色调，逐渐暗淡 | 沉重、反思、不甘 | 低落 | 破碎的像素边框、灰暗色调 |
| **剧情关键时刻** | 紧张、抉择、命运感 | 聚光灯效果，强调选择 | 宿命、戏剧性、庄重 | 平稳 | 巨大的选择像素边框、命运符号 |
| **菜单/界面** | 冷静、准备、沉浸 | 中性灯光，低对比度 | 沉稳、优雅、等待 | 低落 | 精美的像素边框装饰 |

### 像素氛围技巧

| 技巧 | 实现方式 | 效果 |
|------|----------|------|
| **抖动渐变** | 用像素点交错排列模拟光影 | 创造深度感 |
| **调色板切换** | 不同状态使用不同调色板 | 快速改变氛围 |
| **像素粒子** | 用少量像素点创造动态效果 | 节省性能 |
| **分层视差** | 多层背景不同速度移动 | 创造空间感 |

### 氛围转换规则

| 转换 | 调色板变化 | 粒子变化 | 能量变化 |
|------|------------|----------|----------|
| 探索→对话 | 冷→暖 | 减少环境粒子 | 平稳→沉思 |
| 对话→战斗准备 | 暖→冷 | 增加肃杀粒子 | 沉思→平稳 |
| 战斗准备→对战 | 冷→热 | 爆裂式粒子 | 平稳→狂热 |
| 对战→胜利 | 热→金 | 金色粒子爆发 | 狂热→高涨 |
| 对战→失败 | 热→冷 | 粒子消散 | 狂热→低落 |
| 剧情关键 | 中性强调 | 命运符号粒子 | 平稳→紧张 |

## Section 3: 形状语言

### 像素艺术形状原则

**核心理念**：在有限的像素中创造最大的辨识度和氛围。

### 角色轮廓设计

| 角色类型 | 轮廓特征 | 像素辨识度 | 设计测试 |
|----------|----------|------------|----------|
| **玩家角色** | 独特头饰/披风，动态姿态 | 高（缩略图可识别） | 在16x16像素下能否认出？ |
| **盟友NPC** | 友好圆润，装饰性边框 | 中高 | 边框是否体现身份？ |
| **敌人** | 尖锐、不对称、威胁感 | 高 | 是否一眼看出是敌人？ |
| **Boss** | 巨大、复杂、多层轮廓 | 极高 | 是否有压迫感？ |

### 环境几何

| 区域类型 | 几何风格 | 像素表现 | 氛围传达 |
|----------|----------|----------|----------|
| **森林** | 有机曲线，树木轮廓 | 不规则像素簇 | 神秘、自然 |
| **城堡** | 直线、尖角、哥特式 | 规整像素块 | 庄严、压迫 |
| **废墟** | 破碎、不对称 | 残缺像素 | 悲凉、历史感 |
| **神秘空间** | 几何符号、对称 | 精确像素排列 | 宿命、魔法 |

### UI形状语法

| UI元素 | 像素形状 | 装饰性 | 功能 |
|--------|----------|--------|------|
| **普通选择** | 简单矩形边框 | 低 | 区分可交互元素 |
| **重大选择** | 华丽装饰边框 | 高 | 强调重要性 |
| **卡牌** | 圆角矩形+顶部装饰 | 中 | 保持可读性 |
| **血条** | 细长矩形+像素填充 | 低 | 清晰传达信息 |
| **技能图标** | 方形+内部图案 | 中 | 快速识别 |

### 像素艺术关键技巧

| 技巧 | 说明 | 应用场景 |
|------|------|----------|
| **轮廓线** | 用深色像素勾勒轮廓 | 角色、重要物体 |
| **高光点** | 用亮色像素点缀 | 眼睛、武器、魔法效果 |
| **阴影层次** | 用2-3层暗色创造深度 | 角色、环境 |
| **抖动** | 像素点交错排列 | 渐变、光影 |
| **子像素动画** | 用1-2像素变化表现微动 | 呼吸、飘动 |

### 形状语言决策树

```
需要设计新元素？
    ↓
是否是角色/重要物体？
    ├─ 是 → 添加轮廓线 + 高光点
    └─ 否 → 简化形状，减少细节
    
是否需要传达情绪？
    ├─ 是 → 使用对应区域的几何风格
    └─ 否 → 使用中性形状
    
是否是UI元素？
    ├─ 是 → 保持可读性，装饰性边框
    └─ 否 → 融入环境风格
```

## Section 6: 环境设计语言

### 像素环境设计原则

> **用分层和调色板创造深度，用细节暗示故事。**

### 环境层级结构

| 层级 | 像素处理 | 视差速度 | 细节度 |
|------|----------|----------|--------|
| **前景** | 最清晰、最暗 | 最快 | 高（可互动元素） |
| **中景** | 主要游戏区域 | 中等 | 极高（角色活动区） |
| **远景** | 较模糊、较亮 | 最慢 | 低（氛围营造） |
| **天空/背景** | 最模糊、最亮 | 静止 | 极低（颜色渐变） |

### 区域设计规范

#### 森林区域
| 元素 | 像素风格 | 调色板 | 氛围 |
|------|----------|--------|------|
| **树木** | 有机曲线，不规则轮廓 | 大地棕+生命绿 | 神秘、自然 |
| **地面** | 像素噪点纹理 | 暗绿+棕色 | 厚重、古老 |
| **光源** | 萤火虫、月光 | 命运金+月光银 | 梦幻、指引 |
| **隐藏区域** | 发光符文、微光 | 暗影紫 | 神秘、奖励 |

#### 城堡区域
| 元素 | 像素风格 | 调色板 | 氛围 |
|------|----------|--------|------|

| **柱子** | 哥特式尖顶 | 暮色灰 | 高耸、威严 |
| **窗户** | 彩色玻璃效果 | 魔力蓝+命运金 | 神圣、神秘 |
| **火炬** | 动态像素火焰 | 烈焰橙+鲜血红 | 温暖、危险 |

#### 废墟区域
| 元素 | 像素风格 | 调色板 | 氛围 |

### 像素环境技巧

| 技巧 | 说明 | 应用 |
|------|------|------|
| **抖动纹理** | 用像素点创造材质感 | 墙壁、地面、石头 |
| **边缘高光** | 用亮色像素勾勒边缘 | 建筑轮廓、物体边缘 |
| **环境光** | 用颜色渐变表现光照 | 火炬周围、月光下 |
| **粒子氛围** | 用漂浮像素创造氛围 | 灰尘、萤火虫、雪花 |

## Section 8: 资产标准

### 像素资产标准

> **统一标准，高效生产** — 所有像素资产必须遵循以下规范。

### 文件格式规范

| 资产类型 | 格式 | 色彩模式 | 用途 |
|----------|------|----------|------|
| **角色 sprites** | .png | RGBA | Godot导入 |
| **卡牌** | .png | RGBA | Godot导入 |
| **环境 tiles** | .png | RGBA | Godot TileMap |
| **UI元素** | .png | RGBA | Godot Control |
| **动画帧** | .aseprite | — | 源文件保存 |
| **调色板** | .pal | — | Aseprite调色板 |

### 命名规范

```
[type]_[name]_[variant]_[frame].png

示例：
char_player_idle_01.png
card_fireball_rare_01.png
tile_stone_floor_01.png
ui_button_normal_01.png
```

| 前缀 | 类型 | 示例 |
|------|------|------|
| **char_** | 角色 | char_player_walk_01.png |
| **card_** | 卡牌 | card_heal_common_01.png |
| **tile_** | 环境瓦片 | tile_grass_01.png |
| **ui_** | UI元素 | ui_button_hover_01.png |
| **fx_** | 特效 | fx_fireball_01.png |
| **item_** | 物品 | item_potion_01.png |

### 分辨率层级

| 资产类型 | 基础分辨率 | 缩放倍数 | Godot导入设置 |
|----------|------------|----------|---------------|
| **角色** | 32x48 | 4x (128x192) | Nearest |
| **卡牌** | 48x64 | 3x (144x192) | Nearest |
| **环境** | 16x16 | 4x (64x64) | Nearest |
| **UI** | 按需 | 3-4x | Nearest |
| **特效** | 按需 | 2-4x | Nearest |

### Godot导入设置

```yaml
# 所有像素资产使用以下设置
filter: false  # 禁用纹理过滤
mipmaps: false  # 禁用mipmap
compress: 0  # 无压缩
process:
  2d/gen_mipmaps: false
```

### 动画帧规范

| 动画类型 | 帧数 | 帧率 | 循环 |
|----------|------|------|------|
| **idle** | 2-4帧 | 4-8 FPS | 是 |
| **walk** | 4-6帧 | 8-12 FPS | 是 |
| **attack** | 3-5帧 | 12-16 FPS | 否 |
| **skill** | 4-8帧 | 12-20 FPS | 否 |
| **death** | 3-5帧 | 8-12 FPS | 否 |

### 资产生产流程

```
1. 概念设计 → 纸面草图/数字草图
2. 像素化 → Aseprite创建基础形状
3. 调色 → 应用调色板，调整色彩
4. 细节 → 添加高光、阴影、轮廓
5. 动画 → 创建关键帧，调整过渡
6. 导出 → 导出为PNG，应用Godot设置
7. 测试 → 在Godot中测试效果
8. 迭代 → 根据测试结果调整
```
